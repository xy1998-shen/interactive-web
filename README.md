- [总体创意](doc/01-项目原则/总体创意.md)
- [设计语言](doc/01-项目原则/设计语言.md)
- [页面构思](doc/01-项目原则/页面构思.md)
- [交互串联](doc/01-项目原则/交互串联.md)

## 02-开发规范

用于约束技术选型、代码组织、质量标准和开发实现方式。新增功能、重构和验收前都应参考这里。

- [技术方案](doc/02-开发规范/技术方案.md)
- [编码规范](doc/02-开发规范/编码规范.md)

## 维护规则

- 新文档先判断属于哪一类，再放入对应子目录。
- 跨目录引用使用相对路径，避免移动后链接失效。
- 进度类文档记录状态和计划，原则类文档记录长期不轻易变化的判断。
- 设计素材类文档记录可复现的资产来源、规格和版本，不只记录最终结论。
