# 《楚江寻艾》

《楚江寻艾》是一件以湖北屈原故里端午习俗为文化底色的沉浸式交互网页作品。用户沿楚江依次完成拨雾、寻艾、裹粽、击鼓、问诗、和鸣与端午印记收束，在短时间内形成对艾、粽、舟、钟四个端午意象的记忆。

## 打开方式

推荐使用桌面端 Chrome 或 Edge 打开 `index.html`。作品按 PC 横屏体验设计，最佳视口不低于 `960 x 540`，移动端暂不支持。

项目不依赖外链、接口或在线素材，参赛包保留必要运行文件后可直接双击打开。

## 推荐体验路径

1. 入江：轻触江面，拨开江雾，进入楚江端午叙事。
2. 寻艾：顺序浏览悬门、煮汤、入食、佩香，理解艾草如何进入端午日常。
3. 裹青：顺序浏览备料、裹粽、蒸煮、分食，完成粽俗记忆。
4. 听鼓：随自动鼓序列前行，轻点画面加击鼓波与船身惯性。
5. 问诗：轻触三枚字粒，生成一行楚江短诗。
6. 和鸣：沿声波点亮三处回声。
7. 端午印记：点亮艾、粽、舟、钟，完成旅程收束。

## 作品亮点

- 地域明确：围绕湖北屈原故里端午，不做泛化节日页面。
- 交互克制：每幕只有一个核心动作，交互服务叙事而非堆功能。
- 视觉统一：江雾、水纹、艾叶、粽叶、朱砂鼓波、楚辞文字和编钟声波形成连续风格。
- 本地可运行：普通 `<script>` 顺序、本地 UMD 库和相对路径资源，保留 `file://` 兼容。
- 信息闭环：每幕完成后解锁轻知识，尾声汇总四个记忆点。

## 文件说明

- `index.html`：作品入口。
- `css/`：页面样式和桌面端布局。
- `js/`：场景管理、资源加载和 7 幕交互实现。
- `libs/`：本地 PixiJS 与 GSAP。
- `assets/`：运行所需背景、精灵、纹理、字体和标题素材。
- `doc/`：设计、开发、资产和进度文档。
- `design/`：设计过程稿，不建议放入最终提交包。
- `SUBMISSION.md`：提交包范围和评审检查说明。
