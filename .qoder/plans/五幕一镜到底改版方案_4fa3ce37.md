## Task 1：面板切换系统（PanelSwitcher）

**修改文件**：`js/scenes/base-scene.js`、`js/config.js`、`css/style.css`

在 MVPScene 基类中新增轻量面板切换机制，不涉及 Camera/超宽精灵：

```javascript
// 核心数据结构
// panels[]: { bgKey, title, description, buildFn }
// 每个面板是一个独立 PIXI.Container，带自己的背景精灵(viewport尺寸)
// 切换时：当前面板淡出+轻微偏移 → 下一面板淡入（GSAP crossfade ~0.8s）
```

功能清单：
- `initPanels(panelConfigs[])` — 创建面板容器数组，默认显示第1个
- `switchToPanel(index, direction)` — 丝滑切换动画（支持左右滑动+透明度渐变）
- `startAutoPlay(intervalMS)` — 自动轮播（默认5秒/面板）
- `stopAutoPlay()` — 用户手动操作时暂停自动轮播
- 底部圆点指示器（DOM层，CSS实现）
- 支持左右滑动手势切换（移动端）

过渡动画设计：
- 当前面板：alpha 1→0, x 0→-60px, duration 0.8s
- 下一面板：alpha 0→1, x 60px→0, duration 0.8s
- easing: power2.inOut
- 可选方向感（向左/向右滑动）

## Task 5：听鼓场景重写

**修改文件**：`js/scenes/drum-scene.js`、`js/scenes/drum-effects.js`、`js/config.js`

**不使用面板切换**，改为纵深视角的"前进感"：

视觉层次（从后到前）：
```
天空层: 远山/天空（固定不动）
远景层: 河岸树木（缓慢向后滚动，parallax 0.3x）
中景层: 河面水波 + 对岸观众（中速滚动，parallax 0.6x）
近景层: 鼓手背影、船上队员（随鼓点微动，固定位置）
前景层: 船尾栏杆/桨影（固定，微晃动）
```

音频系统（Web Audio API 合成）：
- 入场后自动开始播放（依赖前几幕用户交互已解锁AudioContext）
- 鼓声合成：80Hz正弦波 + 白噪声冲击，envelope快速衰减
- 节奏模式：BPM 70 → 逐渐加速到 130 → 高潮保持4拍 → 回落（约25秒完整周期）
- 每次鼓击同步触发视觉效果：水花粒子、船身微震、鼓波纹扩散

交互：
- 场景入场自动播放鼓声序列（12拍完成）
- 用户可点击画面加入额外鼓击（加速节奏）
- 远景层持续向后滚动营造前进感
- 12拍完成后显示完成文案

素材需求：
- 1张船上第一人称视角背景（含远山天空河面）
- 1张前景层船体框架（PNG透明底）

### AudioContext 解锁策略

在 intro 场景（第1幕）的首次用户交互时创建并 resume AudioContext，存储在全局 `NS.audioCtx`。后续场景直接复用。听鼓场景入场时检查状态，若已 unlocked 则自动播放。

### 性能保障

- 每个面板的背景精灵在不显示时设 `visible = false`（不参与渲染）
- 素材按场景分组懒加载（进入场景前预加载下一场景资源）
- 移动端：图片尺寸可按 DPR 降级（提供 960x540 小图备选）
