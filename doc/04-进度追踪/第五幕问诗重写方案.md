# 第五幕「问诗」重写方案

## 背景

第五幕 `05 问诗` 当前已经具备 MVP 主流程：深青静水、8 个意象字粒、用户选择 3 字、系统按确定性短句库生成 1 条原创短诗。它已经能完成“触字成句”的基础功能，但还没有充分承接第四幕「听鼓」完成态里的“鼓声已渡”和水痕字粒，也缺少诗成之后可停留、可观看、可自然进入第六幕「和鸣」的完成态。

本轮只新增方案文档，不直接改代码、不替换正式逐幕开发文档、不生成素材。目标是先把第五幕剧本重写清楚，作为后续设计和实现的统一依据：第四幕从众人合力的鼓声与竞渡进入高潮，第五幕应让这股动势落回静水，让用户从“随鼓而行”回到“个人求索”，在水中文字里问出一行短诗。

第五幕不是答题游戏，也不是诗词组合器。它应当像龙舟驶过之后，江面慢慢安静，水痕里留下的字被用户轻轻触起：点三枚字，不为判断对错，只为让楚江、艾草、舟影、鼓声与月色在水面回成一句诗。

## 当前问题

1. **整体仍偏 MVP**
   现有文档已经定义了 8 个字粒、选 3 字、确定性匹配短诗和禁止在线随机生成，但更像功能清单。它还没有把“进入、触字、成诗、停留、过渡”写成完整分段剧本。

2. **承接第四幕水痕字粒不足**
   第四幕正式文档已经明确：完成态显示“鼓声已渡”，龙舟水痕中浮出“江”“艾”“舟”“鼓”“风”等字粒，并在点击「入下一幕 →」后扩散为深青静水。第五幕现稿只写“雾青江面转入深青静水”，没有说明水痕和字粒如何进入本幕。

3. **问诗感不够明确**
   “选择任意 3 个字粒”容易被实现成选项按钮或答题格。第五幕应强调“触字问诗”：用户是在听江面回应，不是在从题库里找正确答案。

4. **诗意完成态不足**
   现有文档写到第 3 个字粒被选中后锁定结果，但没有说明短诗生成后江面如何回应、画面如何保持可读、轻知识如何出现，以及用户不立即进入下一幕时画面应停留成什么状态。

5. **转入第六幕的视觉线索偏弱**
   交互串联要求“问诗后，诗字散成声波，声波引出钟影”。现稿只在素材表里列了声波 SVG，没有把诗字散为声波、远处青铜钟影出现的过渡写进剧本。

6. **轻知识表达偏直白**
   “屈原诗意与端午记忆相连”方向正确，但第五幕不宜出现百科卡片。轻知识应通过“触字显义”出现，文案短、轻、可重看，不阻塞下一幕。

## 重写目标

- 把第五幕定位为全篇从“动”回到“缓”的情绪转折：第四幕是众人合力、鼓声渡江；第五幕是个人触字、静水问诗。
- 明确承接第四幕完成态：`鼓声已渡`、长水痕、水痕字粒是第五幕入场的直接来源。
- 保留一个主动作：轻触字粒；其他诗句匹配、江面回应和过渡声波都由系统自动完成。
- 让用户感到“字从水下被问起”，而不是完成“选 3 个正确答案”的任务。
- 视觉上延续深青静水、雾白水纹、竹影暗纹和少量青铜金，不让画面变成黑夜、题卡或诗词测验界面。
- 诗成之后有可停留的完成态：短诗清楚、字粒安静、轻知识可触发、底部入口出现但不自动硬切。
- 过渡时让诗字笔画散成圆形声波，远处青铜钟影浮现，自然引出第六幕「和鸣」。
- 保持静态站约束：普通 `<script>` 顺序、UMD 全局对象、`file://` 可打开，不引入远程资源。

## 新版剧本

### 1. 水痕入诗

第四幕完成后，画面停在龙舟驶过的长水痕上，完成短句“鼓声已渡”仍在水面低透明保留。用户点击「入下一幕 →」后，不硬切新场景，而是让长水痕向四周慢慢扩散，朱砂余色退成淡金和雾白，江面速度明显放慢。

画面从竞渡水路转为深青静水。第四幕水痕中的 3 到 5 枚字粒先被带入本幕，例如“江”“艾”“舟”“鼓”“风”。它们像从水下浮起，位置仍沿着上一幕水痕方向散开，不立刻排成规则字阵。

入场短句淡入：

> 鼓声入水，字从痕起。

系统反馈是“动势变静”：鼓波余韵消失，水面只剩细小涟漪，背景雾气速度减慢。入场约 1200ms，至少比第四幕完成过渡慢，让用户感到节奏沉下来。

### 2. 字粒浮现

水痕稳定后，剩余意象字粒从静水中继续浮现，最终形成 8 个可触字粒：

> 江、艾、舟、鼓、风、月、青、钟

字粒不排成横平竖直的答题选项，而是以疏密有致的水面分布出现：中心偏左或偏中为“江”“舟”“风”，靠近竹影或暗纹处为“月”“青”，靠近远处声波方向为“钟”。字粒之间留足空白，不能遮挡短句和底部入口区域。

每个字粒只做轻微正弦漂浮和透明度呼吸。鼠标悬停时，字粒周围出现一圈很淡的雾白水纹；点击前不显示“可选”“正确”“错误”等现代 UI 提示。

动作提示使用底部江面题签：

> 轻触三字，问一行诗

用户此时可以自由选择字粒。系统只通过水纹和微光提示可触，不主动弹出说明卡。

### 3. 点字成句

用户点中第 1 个字粒时，该字轻亮并向上漂浮一小段，原位置留下一圈 600ms 至 900ms 的细水纹。被选中字粒不脱离水面太远，仍像浮在江上，不变成标签或棋子。

用户点中第 2 个字粒时，两枚已选字之间出现极淡水线，像江面把两个意象暂时连起来。未选字仍保持低透明漂浮，不消失、不灰掉，避免产生“已排除选项”的答题感。

用户点中第 3 个字粒时，系统锁定本次结果。三枚字粒缓慢靠近画面中央偏侧，按预设短句的阅读方向生成 1 条原创短诗。匹配逻辑仍保持确定性：同一组字永远返回同一条短诗，不接入在线生成，不做随机拼句。

如果用户在未满 3 字前再次点击已选字，该字轻轻落回原位并取消选中；如果已经选满 3 字，用户可以点击“重选”轻题签让字粒散回水面，重新触字。取消和重选都不出现失败或错误提示，只表现为水纹回落。

### 4. 江面回应

短诗生成后，背景雾气进一步放慢，短诗在水纹中稳定显示。它可以竖排，也可以随水痕略微弯曲，但只显示 1 条，不铺满屏幕，不做大画轴或现代字幕条。

推荐完成短句：

> 字从水起，风过楚江。

短诗下方或旁侧出现极轻关键词：

> 屈原 · 端午 · 楚江

用户悬停或点击关键词时，关键词向外扩散一圈水纹，并补全轻知识：

> 屈原诗意与端午记忆相连，是楚江旅程的精神底色。

轻知识停留 3 至 5 秒后淡出，关键词保留，用户可再次触发。不点击轻知识不阻塞进入下一幕。

完成态需要可停留：短诗、3 枚已选字、未选字低透明水影、关键词和底部「入下一幕 →」入口都保持可读。如果用户不点击入口，画面不自动跳走。

### 5. 诗成过渡

用户点击「入下一幕 →」后，短诗不直接消失，而是先拆成细小笔画光点。笔画光点从文字边缘散开，沿水面形成一到两圈圆形声波。

圆形声波使用青铜金和雾白低透明线条，扩散速度慢于普通点击水纹，约 1200ms 至 1800ms。声波向远处扩散时，8 个字粒依次变淡，只留下少量笔画碎片随波前行。

声波远端浮现抽象青铜钟影，画面进入第六幕「和鸣」的前奏。过渡短句可以短暂出现：

> 诗声入水，远钟将鸣。

这句只作为过渡回声，不替代第六幕正式短句“金声入水，楚江和鸣。”

## 交互与动效规则

- 主交互对象为 8 个水面字粒，点击范围应略大于视觉文字，保证用户不用精确点中笔画。
- 成诗需要最少 3 个字、最多 3 个字。未满 3 个时只保持“轻触三字，问一行诗”的动作提示，不生成半成品诗句。
- 第 3 个字选中后锁定结果，不继续追加第 4 个字。
- 已选未成诗时，用户再次点击已选字可取消；取消反馈为字粒轻落回水面和水纹淡出。
- 已成诗后，提供“重选”轻题签；点击后短诗淡出，3 枚已选字回到水面，重新进入可选状态。
- 不出现分数、倒计时、正确、错误、失败、通关、连击、评级、排行榜等游戏化表达。
- 字粒不能做成按钮网格、答题卡、棋盘或卡牌。它们应像浮在水面的字，而不是可点击选项。
- 点击反馈控制在 220ms 左右：字亮、微上浮、水纹生成同时发生；不要强缩放、爆闪或弹跳。
- 字粒漂浮幅度要小，避免影响可读性。建议垂直浮动不超过 6px 至 10px，周期 1600ms 至 2400ms。
- 短诗生成时长控制在 900ms 至 1400ms，足够慢，和第四幕短促鼓波形成对比。
- 完成过渡由单一完成函数接管：先稳定显示短诗，再显示入口；点击入口后才播放诗字散成声波。
- 轻知识使用“触字显义”：关键词触发水纹和短句，不使用弹窗百科，不阻塞下一幕。
- 如果字粒配置为空或短诗库缺项，后续实现应直接跳过对应效果或使用明确兜底文案，不能出现空白文字、`undefined` 或运行时报错。

- 诗成声波过渡纹理：如 SVG 圆环过于干净，可新增低透明水波纹理，方向为“青铜金细线声波、融入深青水面、克制、无舞台光效”。

### 风格边界

- 画面应比第四幕安静，但不能全黑、阴森或祭祀恐怖化。
- 深青水面要保留雾白、浅水青和少量青铜金，避免变成纯黑夜景。
- 竹影或竹简暗影只做层次，不做知识卡背景，不抢字粒。
- 字粒大小要清楚可读，不能过度书法化导致辨认困难。
- 短诗不做满屏大字，不做卷轴弹窗，不做现代输入框或生成器界面。
- 青铜金只用于关键词、声波和进入第六幕的远钟线索，不大面积铺金。

## 与前后分幕的衔接

### 承接第四幕「听鼓」

- 第四幕完成短句“鼓声已渡”是第五幕入场前的最后可读文字。
- 第四幕长水痕不是消失，而是扩散成第五幕深青静水。
- 第四幕水痕中浮出的“江”“艾”“舟”“鼓”“风”等字粒先进入第五幕，再补足“月”“青”“钟”形成完整字池。
- 第四幕的朱砂红鼓波在第五幕退成淡金和雾白，只保留余韵，不延续竞渡动感。
- 用户从第四幕点击「入下一幕 →」后，应看到“水痕变静水、字粒重新浮现”的物象过渡，而不是突兀切到独立选字页面。

### 引出第六幕「和鸣」

- 第五幕成诗后，诗字不是直接退出，而是拆成笔画光点。
- 笔画光点沿水面扩散成圆形声波，呼应第六幕编钟声波。
- “钟”字可以在过渡时最后一次轻亮，远处青铜钟影随声波浮现。
- 第六幕正式短句仍使用“金声入水，楚江和鸣。”；第五幕只用“诗声入水，远钟将鸣。”作为过渡回声。
- 如果用户不点击入口，画面保持第五幕完成态；只有主动点击入口才进入声波过渡。

## 实现经验吸收

第五幕后续实现应吸收第四幕落地时暴露出的 Pixi / GSAP 状态经验：

- Pixi / GSAP 动画必须基于进入场景时记录的 base 值，例如字粒 `baseX`、`baseY`、`baseAlpha`、`baseScaleX/Y`。漂浮、选中上浮、取消回落和完成成诗都从 base 值计算，避免多次重选后位置漂移。
- 同一对象的同一属性在新动画接管前先 `killTweensOf`，尤其是字粒 `x/y/alpha`、`scale.x/y` 和短诗容器透明度。
- Pixi 缩放使用 `scale.x/y`，不要使用 DOM 风格的 `scaleX/scaleY`。
- 字粒、短诗库、关键词、轻知识文案等配置允许为空；入口函数要先做防御检查，空数组不取模，空文案不进入 `PIXI.Text`。
- 循环创建水纹、笔画光点或临时字粒时，`onComplete` 必须引用当前局部对象并销毁，避免闭包引用错乱。
- 完成态由单一函数接管，例如 `completePoem()`：锁定选择、生成短诗、设置完成短句、显示关键词和入口都由它统一处理，避免最后一帧 hint 闪烁或重复入口。
- 恢复完成态、快速重选、快速切换场景时，`knowledgeText`、`keywordText`、`nextButton`、`poemText` 等对象调用前必须判断存在。
- 轻知识仍用“触字显义”或“随物解锁”的轻量模型，不新增弹窗百科，也不阻塞进入下一幕。
- 保持 `file://` 兼容：不使用 ES Module、fetch 本地文件或远程字体 / 素材；继续依赖本地 UMD 脚本和静态资源清单。
- 不引入远程资源、在线诗句生成或联网 API。短诗库必须本地、确定、可审核。

## 验收要点

- 入场能自然承接第四幕：能看见“鼓声已渡”后的水痕扩散和字粒入诗，而不是突兀进入独立选字页。
- 第五幕节奏明显慢于第四幕，画面从竞渡动感回到深青静水。
- 用户感受到的是“问诗”，不是答题、配对、闯关或诗词小游戏。
- 8 个字粒清晰可读，漂浮和水纹不影响辨认。
- 字粒分布有水面留白，不遮挡短诗、关键词和底部入口。
- 最少 / 最多 3 字成诗；未满 3 字不生成短诗，满 3 字后不追加第 4 字。
- 取消选择和重选都温和，不出现错误、失败、扣分或倒计时。
- 同一组选字生成同一条短诗，不接入在线随机生成。
- 短诗只显示 1 条，文案短、轻、可读，不铺满屏幕。
- 轻知识通过关键词触发，停留后淡出，可再次触发，不使用百科卡片。
- 完成态能停留：短诗、字粒、关键词和入口都可读，用户不点击入口时不自动跳转。
- 点击入口后，诗字能自然散成声波，引出远处青铜钟影和第六幕「和鸣」。
- 视觉符合设计语言：雾青江光、深青静水、竹影、水纹、少量青铜金，不全黑、不高饱和、不游戏化。
- 后续实现必须保持 `file://` 本地打开可用，不依赖远程素材、远程字体、模块加载或联网接口。
