# 第三幕「裹青」打磨计划

## 背景

第三幕 `03 裹青` 当前已经具备 MVP 主流程：左叶合拢、右叶合拢、拖动青线收束，并能继续进入后续分幕。

本轮不新增大型玩法，目标是把现有交互从”可玩”打磨到更有场景感和完成度：背景从通用江景换为”室内木桌包粽、窗外望江”的专属场景，三步动作更清楚，完成态更像粽形，米粒与字粒聚合更有手作感，轻知识出现方式更贴合”束青线末端小光点”的设定，并自然引出第四幕”听鼓”。

## 当前问题

1. **完成态辨识度不足**
   现有收束主要依赖两片叶子靠拢和旋转，完成后容易看成叶片重叠，不一定能一眼识别为粽形。

2. **粒子表现偏弱**
   当前米粒粒子已有聚合雏形，但缺少“米粒 / 字粒被包入叶心”的明确视觉层次，手作感和祝愿被包入的叙事还不够强。

3. **轻知识触发不够贴合设定**
   逐幕文档要求轻知识通过束青线末端光点出现。当前完成后直接展示通用知识文本，缺少“线尾光点”这个小仪式。

4. **进入第四幕的视觉承接不足**
   文档设定为粽形在水面旋转，轮廓逐渐变成龙舟船头或水纹方向线，再进入“听鼓”。当前更多依赖按钮进入下一幕，画面衔接还可以加强。

## 改进目标

- 保留固定三步：合左叶、合右叶、束青线。
- 拖动不到位只做温和回弹和提示，不出现失败惩罚。
- 完成态必须明确形成简化粽形。
- 米粒和字粒从叶面、水面周围向粽形中心聚合。
- 轻知识通过线尾或粽形边缘的小光点出现。
- “入下一幕”前给出粽形旋转、水纹或船头方向线索，承接第四幕鼓点与龙舟。
- 保持静态站约束：普通 `<script>` 顺序、UMD 全局对象、`window.ChuJiang` 命名空间，不改 ES Module。

## 实现方案

### 1. 强化第三幕状态机

主要修改 `js/scenes/wrap-scene.js`。

- 继续使用 `state.step` 控制流程：
  - `0`：等待左叶。
  - `1`：等待右叶。
  - `2`：等待束线。
  - `3`：完成收束。
- 左叶未完成前，右叶光点不可触发。
- 右叶光点在 step=0 时 alpha=0.3、cursor=default，step=1 时 fadeIn 到正常态并切 cursor=grab，明确引导顺序。
- 光点拖到目标区后自动播放折叠动画，避免用户做自由折叠。
- 未命中目标区时回弹到起点，并保留温和提示。

### 2. 优化粽形完成态

主要修改 `js/scenes/wrap-scene.js` 和 `js/config.js`。

- 收束结束后，让两片叶子继续轻微靠拢、缩放和旋转。
- 叶片降至 alpha 0.2–0.3，保留叶脉纹理透出感。
- `sprite-zongzi` 从 alpha 0 → 0.9 淡入（duration 0.5s），叠加在叶片上方作为完成态轮廓。
- `sprite-zongzi-thread` 紧随其后淡入（delay 0.2s），作为束线视觉强化。
- 时序：束线拖完 → 叶子靠拢并降透明度 → zongzi fadeIn → zongziThread fadeIn。
- 不移除叶片 sprite，形成”由叶折成粽”的连续感。

### 3. 米粒和字粒聚合（两阶段）

不新增图片素材，使用 Pixi 动态生成。

**阶段 A：交互过程中的叶心散布**

- `buildWrap` 初始化时在两叶之间 spawn 8–12 个静态米粒点。
- 米粒 alpha 0.3–0.4，微幅上下浮动（gsap yoyo），模拟分镜中叶心金色碎光。
- 左叶合拢时，这些粒子跟随叶片 x 偏移向中心收拢。
- 右叶合拢时进一步聚拢到 centerX 附近。

**阶段 B：束线完成后的聚合**

- 束线完成后，从粽形外圈（60–120px）额外 spawn 新粒子，向中心聚合。
- 阶段 A 剩余粒子与阶段 B 粒子合并，一起收入粽形内部后淡出。
- 粒子颜色：米白 `0xf1e8d2`、浅青 `0xd8ded1`、淡灰绿 `0x9eaea3`。

**字粒**

- 字池：`”愿””安””艾””江””青”`，每次从中随机选 3 个显示。
- 字号 16px，颜色淡金 `0xc8a45d` alpha 0.6，与缠线/目标光点同色系。
- 字粒混在阶段 B 粒子中一起聚合，不单独强调。

### 4. 轻知识光点（自动展开，不要求点击）

主要修改 `js/scenes/wrap-scene.js`。

- 束线完成后，在粽形右下角冒出一个小光点（radius 6，淡金色）。
- 光点 pulse 闪烁 2 次（约 1s），然后扩散消失。
- 光点消失的同时调用 `finish(true)`，触发 base-scene 的 knowledge 文案自动淡入。
- 与 1、2 幕保持一致：knowledge 始终自动淡入，不需要用户额外点击。
- 光点只是增加”线尾仪式感”的视觉装饰，不阻塞任何流程。
- 轻知识文案沿用 base-scene 已配置的：
  `粽叶裹米，既是节令食物，也承接了江水与草木的清香。`

### 5. 衔接第四幕「听鼓」

- 完成态 zongzi sprite 慢速旋转（rotation 0 → 0.15，duration 2s），暗示水面流动。
- 画面右下 spawn 2 个淡红色 expanding ring（颜色 `0xa94f3f`，与第四幕鼓的 ringColor 一致），alpha 0.2 → 0 扩散消失，暗示远处有鼓点节奏。
- 不做粽形 → 船头的形变，第四幕本身是全新场景，无需精确形态承接。
- “入下一幕”入口仍由 DOM 层 `finish()` 触发，点击后进入第四幕。
- 不自动跳转，保留用户确认。

## 涉及文件

- `js/config.js`：第三幕新增参数（zongzi fadeIn duration、叶片目标 alpha、粒子阶段 A 数量、字粒配置、光点参数、旋转角度、水纹 ring 颜色等）。

## 验收标准

### 静态检查

- `node --check js/scenes/wrap-scene.js`
- `node --check js/config.js`

### 交互检查

- 初始态右叶光点半透明、不可拖拽。
- 左叶拖到中心后完成第一折，右叶光点 fadeIn 变为可拖拽。
- 左叶未到中心时回弹，提示仍清楚。
- 右叶完成后才出现束青线。
- 束青线拖不到底时回到起点或提示继续拖动。
- 束青线拖到底后：叶片降透明度 → zongzi sprite 淡入 → thread sprite 淡入 → 形成明确粽形。
- 粽形右下出现光点闪烁 → 自动展开 knowledge 文案。
- knowledge 展示后，”入下一幕”按钮可用，不被阻塞。
- 点击”入下一幕”能进入”听鼓”。

### 视觉检查

- 初始态：两叶展开，叶心有 8–12 个微浮动米粒点，贴近分镜 01。
- 交互中：叶片合拢过程中，米粒跟随收拢，贴近分镜 02。
- 完成态：粽形清晰 + 金线 + 粒子聚合消失 + 慢旋转 + 远处淡红水纹，贴近分镜 03。
- 字粒（3 个字）混在粒子中聚合，不抢主视觉。
- PC 横屏和移动窄屏下，文字、按钮、知识点不互相遮挡。
- `file://` 和本地静态服务两种打开方式都能进入并完成第三幕。

## 实现时序总览

```
buildWrap
  ├─ 叶片 + 目标光点 + 拖拽光点（右叶 alpha=0.3）
  ├─ 阶段 A 米粒 spawn（叶心 8-12 颗，微浮动）
  │
  ├─ [用户拖左叶] → completeStep("left")
  │     左叶靠拢 + 阶段 A 粒子跟随收拢
  │     右叶光点 fadeIn
  │
  ├─ [用户拖右叶] → completeStep("right")
  │     右叶靠拢 + 阶段 A 粒子进一步聚拢
  │     显示束线
  │
  ├─ [用户拖束线到底] → completeWrap
  │     叶片 alpha → 0.25
  │     zongzi sprite fadeIn 0.5s
  │     zongziThread sprite fadeIn (delay 0.2s)
  │     阶段 B 粒子 + 字粒 spawn → 向中心聚合 → 淡出
  │     光点出现 → pulse 2次 → 消失
  │     finish(true) → knowledge 淡入 + "入下一幕"按钮
  │     zongzi 慢旋转 + 远处 2 个淡红 ring 扩散
  │
  └─ [用户点击"入下一幕"] → 进入第四幕
```

## 默认取舍

- 不做真实包粽教程，只保留抽象三步仪式。
- 不新增依赖。
- 新增一张专属背景 `bg-wrap-table.webp`（室内木桌+窗外江景），其余复用现有 `sprite-zongzi`、`sprite-zongzi-thread`、`sprite-leaf-left/right`。
- 不改动 `base-scene.js`，knowledge 文案和 finish 逻辑原样复用。
- 不顺手重构其他分幕。
- 若拖拽手感和视觉效果冲突，优先保证流程清楚、不会卡住。
- 束线完成后可选用 Graphics 画 2–3 圈弧线环绕粽形作为缠线装饰（视实现效果决定是否保留）。
