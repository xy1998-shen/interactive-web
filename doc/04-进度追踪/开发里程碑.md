# 《楚江寻艾》开发里程碑

## 一、总体节奏

14 天开发周期，分 4 个阶段。核心策略：**梯队优先级制**，确保任何时间点都有可提交的完整作品。

```
第 1-3 天   ████░░░░░░░░░░  基础骨架 + 入江场景
第 4-7 天   ████████░░░░░░  核心 5 幕贯通（MVP 可提交）
第 8-11 天  ████████████░░  全 7 幕 + 变形过渡
第 12-14 天 ██████████████  打磨 + 测试 + 提交
```

## 二、阶段一：基础骨架（第 1-3 天）

### 第 1 天：项目搭建

| 任务 | 产出 | 验收标准 |
|------|------|----------|
| 初始化项目结构 | 完整目录骨架 | 目录清晰、文件就位 |
| 引入 PixiJS + GSAP | 库文件可用 | 控制台无报错 |
| PixiJS 全屏画布 | 暗色全屏渲染 | #101C1B 背景，resize 自适应 |
| 基础场景管理器 | SceneManager | 可切换空白场景 |
| DOM 浮层结构 | 文字层可显示 | 标题文字可见 |

**当天素材需求**：bg-river-dark、bg-fog-layer、tex-water-ripple、noise 纹理

### 第 2 天：入江场景

| 任务 | 产出 | 验收标准 |
|------|------|----------|
| 雾气置换滤镜 | 雾有机流动 | 视觉柔和、不卡顿 |
| 遮罩交互（拖动拨雾） | 拖动可清除雾气 | 响应 < 16ms，自然感 |
| 标题浮现逻辑 | 标题随拨雾出现 | 时机正确、动画优雅 |
| 水纹扩散效果 | 指针附近有涟漪 | 不抢视觉焦点 |
| 场景完成判定 | 60%面积触发完成 | 过渡自然 |

### 第 3 天：基础设施

| 任务 | 产出 | 验收标准 |
|------|------|----------|
| 音频引擎（4 音色） | AudioEngine 可用 | 各音色可独立触发 |
| 资源加载器 | AssetLoader | 支持按场景分组加载 |
| 预加载逻辑 | 场景 N+1 静默加载 | 切换时无等待 |
| 入江场景添加音效 | 水声环境音 | 拨雾有轻微音效反馈 |

**阶段一验收**：打开页面 → 流动雾气覆盖暗色江面 → 拖动拨雾 → 标题浮现 → 水纹扩散 → 环境水声 → 60fps。

## 三、阶段二：核心场景（第 4-7 天）

### 第 4 天：寻艾场景

| 任务 | 产出 | 验收标准 |
|------|------|----------|
| 拖艾入户交互 | 艾草拖动、窗前/门楣吸附 | 拖动顺畅、目标清晰、吸附反馈明确 |
| 轨迹粒子效果 | 绿色香气轨迹 | 拖动路径有淡绿粒子淡出 |
| 目标吸附机制 | 窗前 / 门楣目标区域 | 接近时自动吸附锁定 |
| 挂艾人物动画 | 窗前抬手挂艾 | 吸附后触发、动作清晰 |
| 知识点展示 | 短句浮现 | 挂艾完成后显示文案 |
| 艾草音效 | 沙沙声 | 拖动时触发 |

**当天素材需求**：sprite-mugwort、bg-mugwort-village

### 第 5 天：听鼓场景

| 任务 | 产出 | 验收标准 |
|------|------|----------|
| 鼓面点击区域 | 可点击响应 | 命中区域合理 |
| 朱砂鼓波效果 | 红色圆环扩散 | 每点一次生成一环 |
| 龙舟推进 | 龙舟向右移动 | 每点推一步，有缓动 |
| 连击加成 | 快速点击增强 | 400ms 内连击有反馈 |
| 鼓声合成 | 节奏鼓点 | 80Hz 短促有力 |

**当天素材需求**：sprite-drum、sprite-boat

### 第 6 天：问诗场景 + 尾声场景

| 任务 | 产出 | 验收标准 |
|------|------|----------|
| 8 字浮动精灵 | 文字缓动 | 正弦波浮动自然 |
| 选字交互+确定性匹配 | 选 3 字出诗 | 任何组合都有确定输出 |
| 诗句纵向展示 | 浮现动画 | 时机正确 |
| 印记汇聚动画 | 5 元素向中心收束 | 2s 内完成，有层次 |
| 重游按钮+状态重置 | 主题化入口 | 重游后体验完整 |

### 第 7 天：场景串联

| 任务 | 产出 | 验收标准 |
|------|------|----------|
| 7 幕串联流程 | 入江→寻艾→裹青→听鼓→问诗→和鸣→尾声 | 连续可玩不断裂 |
| 过渡动画（渐隐渐显） | 800ms 过渡 | 平滑不突兀 |
| 江流进度线 | 7 节点进度指示 | 完成时点亮 |
| 题签 UI | 各幕标签 | 风格统一、不抢焦点 |
| 端到端测试 + file:// 验证 | 全流程通过 | 断网双击可用 |

**阶段二验收（MVP）**：7 幕完整可进入、可完成、可收束 → 每幕有交互或轻交互+知识点 → 过渡平滑 → 进度可见 → 可重游 → file:// 可用。**此时已具备提交资格。**

## 四、阶段三：完整体验（第 8-11 天）

### 第 8 天：和鸣场景

| 任务 | 产出 | 验收标准 |

**当天素材需求**：sprite-bell

| 任务 | 产出 | 验收标准 |
|------|------|----------|

| file:// 再次验证 | 新增场景无异常 | 断网可用 |

**当天素材需求**：bg-bamboo

### 第 10 天：裹青场景

| 任务 | 产出 | 验收标准 |
|------|------|----------|
| 粽叶精灵布局 | 左右对称 | 初始位置清晰 |
| 拖动折叠（或降级方案） | 可操作完成 | 手感流畅不卡顿 |
| 米粒引力效果 | 粒子聚向中心 | 配合折叠节奏 |
| 文字聚合 | 诗字嵌入粽形 | 有意象感 |

**关键决策点**：第 10 天下午评估折叠交互手感。如不满意，立即切换降级方案（单步拖拽+自动动画）。不在此处纠结超过 4 小时。

### 第 11 天：物象变形过渡

| 任务 | 产出 | 验收标准 |
|------|------|----------|

**当天素材需求**：全部 SVG 矢量路径、tex-pattern-duanwu

**阶段三验收**：7 幕全部可玩 → 至少 3 个变形过渡 → 完整故事弧线 → 情绪节奏正确。

## 五、阶段四：打磨与提交（第 12-14 天）

### 第 12 天：视觉打磨

| 任务 | 说明 |
|------|------|
| 粒子调优 | 各场景粒子密度、大小、速度微调 |
| 呼吸动画 | 背景 0.5% 缓慢缩放（8s 周期） |
| 主题光标 | 水面水纹 / 拖动叶影 / 击鼓红波 / 触钟金波 |
| 进度线完善 | 7 节点完整、金色填充动画 |
| 题签排版 | 字距、进入时机、淡入淡出 |
| 色彩审计 | 逐幕检查是否符合 5 色系统 |

### 第 13 天：音频与过渡打磨

| 任务 | 说明 |
|------|------|
| 环境音渐变 | 场景间音量交叉淡化 500ms |
| 鼓点动态 | 连击时音量+音高微增 |
| 编钟混响 | 添加卷积混响节点 |
| 声画同步 | 确保视觉和音频在 1 帧内对齐 |
| 剩余过渡 | 补完或增强未做的变形过渡 |
| 静音测试 | 关闭声音后视觉体验仍完整 |

### 第 14 天：测试与提交

| 任务 | 说明 |
|------|------|
| 分辨率测试 | 1366×768、1920×1080、2560×1440 |
| 浏览器测试 | Chrome、Firefox、Safari |
| 离线测试 | 断网后 file:// 协议打开验证 |
| 性能测试 | 确认 60fps、无内存泄漏 |
| 代码清理 | 删 console.log、source map |
| 文件大小审计 | 按比赛要求压缩运行必需文件，排除过程材料 |
| 打包提交 | zip 压缩、在新目录解压验证 |
| README / SUBMISSION | 面向评委说明作品、打开方式、推荐路线、提交包边界和不支持范围 |

**阶段四验收（最终）**：断网解压即用 → PC 横屏 7 幕完整 → 声画同步或静音体验完整 → 进度清晰 → 色彩统一 → 提交包范围清晰。

## 六、每日工作节奏建议

```
上午（3-4h）：核心编码/交互实现
下午（2-3h）：素材生成+后处理 / 效果调试
晚间（1-2h）：端到端测试 / 修复问题 / 规划次日
```

素材生成和编码可并行：AI 生成图片等待时可以写代码，代码编译等待时可以调素材。

## 七、关键决策时间点

| 时间 | 决策 | 如果不满意 |
|------|------|-----------|
| 第 2 天末 | 拨雾效果是否达标？ | 简化为圆形扩散清除（降低但保底） |
| 第 7 天末 | MVP 是否可提交？ | 如果 5 幕不稳，第 8-9 天优先修复而非加新幕 |
| 第 10 天下午 | 裹青折叠是否可用？ | 立即切降级方案，不超过 4 小时 |
| 第 11 天末 | 变形过渡效果如何？ | 不好的改回增强渐隐，不强求 |
| 第 13 天末 | 是否还有严重问题？ | 第 14 天只修 bug 不加功能 |

## 八、团队协作建议（如有协作者）

如果有设计师或其他协作者配合：

| 角色 | 主要职责 | 协作时机 |
|------|----------|----------|
| 开发（主力） | 编码、交互实现、音频合成 | 全程 |
| 视觉（AI+人工） | 素材生成、后处理、色彩校正 | 第 1-8 天密集 |
| 文案 | 诗句撰写、知识点定稿 | 第 1-3 天完成 |
| 测试 | 多设备/浏览器验证 | 第 12-14 天 |

如果是一个人完成：按"上午编码、下午素材"的节奏交替推进，避免单一任务疲劳。

## 九、一句话里程碑准则

第 7 天有一个"能交的作品"，后 7 天让它变成"能赢的作品"。
