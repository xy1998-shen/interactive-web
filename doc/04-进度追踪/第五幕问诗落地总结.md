# 第五幕「问诗」落地总结

## 背景

第五幕已经从 MVP 版“点击 3 个字粒生成短诗”推进到正式剧本口径：承接第四幕“鼓声已渡 / 水痕字粒”，进入深青静水中的触字问诗；用户每轮轻触 3 字生成 1 条短诗，成诗后可以“再问一诗”，也可以主动进入第六幕“和鸣”。

本轮目标不是做视觉验收，而是把剧本、正式逐幕文档、实现逻辑、素材接入和关键状态防护对齐。

## 已完成

- 正式逐幕文档 `doc/05-逐幕开发文档/05-问诗.md` 已更新为新剧本：
  - 水痕扩散入诗。
  - 每轮 3 字成诗。
  - 支持“再问一诗”多轮组诗。
  - 成诗呈现口径更新为“三字成脉，诗句入轴”。
  - 关键词触发轻知识。
  - 诗字散成声波，引出第六幕“和鸣”。
- 交互串联文档 `doc/01-项目原则/交互串联.md` 已同步，移除旧的“一次锁定”描述，改为每轮 3 字和“再问一诗”。
- 第五幕主背景已接入无字人物版本：
  - 运行资源：`assets/backgrounds/bg-poem-question-figure.png`
  - 备用背景：`assets/backgrounds/bg-poem-water.webp`
  - 源图归档：`assets/source/poem-question-figure-generated-20260527.png`
  - v2 背景已去掉内嵌中文大字，避免和前景交互字粒重叠。

  - 每轮成诗后，入场短句和字阵先退场，3 枚选中字转为诗轴上方小字，当前短诗在诗轴中两行排版，再显示关键词、“再问一诗”和下一幕入口。
  - 轻知识触发增加 700ms 防抖，避免 hover / click 快速重复触发闪烁。
  - 点击下一幕入口后，诗句淡出、声波扩散、远处编钟预显，再切到第六幕。
- `js/config.js` 已补充第五幕集中配置：
  - 入场、完成、再问、轻知识和过渡文案。
  - 水痕承接字、成诗水痕、远钟预告、声波过渡参数。
  - 轻知识触发冷却时间。
- `js/constants.js` 第五幕加载 `poemWaterPoet / poemFigureBg / poemWater / bell`；`poemWaterPoet` 为当前默认背景，`poemFigureBg` 和 `poemWater` 保留为备用背景。
- `js/asset-loader.js` 已调整 `file://` 策略：优先加载真实本地图片，加载失败再走 fallback canvas，避免双击本地打开时直接丢掉真实背景。

## 关键决策

- **远景诗人题诗背景正式采用 v1**：`bg-poem-water-poet.webp` 作为运行主背景；无字人物背景 `bg-poem-question-figure.png` 保留为 fallback 1。
- **多轮组诗优先于一次完成**：第 3 个字只完成本轮，不结束第五幕；用户可以继续“再问一诗”。
- **每轮只突出当前短诗**：上一轮诗句淡成水痕回声或淡出，不累积成诗墙，避免信息堆叠。
- **成诗不是普通结果文本**：成诗态会收起字阵和入场文案，只显示横向诗轴；完整短诗进入诗轴两行排版，选中字作为“种子字”显示在轴首。
- **非游戏化表达保持不变**：不出现分数、倒计时、正确、错误、失败、通关或评级。
- **竹影叠层撤下**：`bg-bamboo.webp` 有矩形底，在新人物背景上会形成右侧大方块；第五幕不再叠加该素材。
- **水痕承接为视觉模拟**：由于场景切换会销毁第四幕 Pixi 容器，第五幕暂不直接复用第四幕对象，而是在第五幕入场重新绘制水痕和承接字，保持叙事连续。

3. **水痕入诗只停留在文案**

   - 现状：已补 `createPoemBellPreview()`，点击下一幕时编钟剪影淡入。

   - 现状：已用无字 v2 覆盖 `bg-poem-question-figure.png` 并重新接入运行；旧 v1 问题已解决。

8. **短诗呈现方式偏普通结果文本**
