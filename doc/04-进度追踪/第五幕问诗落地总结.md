# 第五幕「问诗」落地总结

## 背景

第五幕已经从 MVP 版“点击 3 个字粒生成短诗”推进到当前正式口径：承接第四幕“鼓声已渡 / 水痕字粒”，通过宽幅楚江背景慢速平移落入静水问诗；用户每轮在宣纸诗笺上轻触 3 字，以毛笔圈痕完成选字，生成 1 条五言二句短诗。成诗后可以“重新圈字”，也可以主动进入第六幕“和鸣”。

本轮目标不是做视觉验收，而是把剧本、正式逐幕文档、实现逻辑、素材接入和关键状态防护对齐。

## 关键决策

- **宽幅楚江背景为当前主方案**：`poem-riverside-market.png` 作为运行主背景；`bg-poem-water-poet.png`、`bg-poem-question-figure.png` 和 `bg-poem-water.webp` 保留为 fallback。
- **平移结束后再提示**：左上标题区不再放重复说明，底部动作提示只在画面定格后出现。
- **宣纸承载优先于水面散字**：字写在纸上更符合问诗气质，也能让毛笔圈字和成诗结果统一在同一载体内。
- **多轮组诗优先于一次完成**：第 3 个字只完成本轮，不结束第五幕；用户可以继续“重新圈字”。
- **每轮只突出当前短诗**：上一轮诗句淡成水痕回声或淡出，不累积成诗墙，避免信息堆叠。
- **成诗不是普通结果文本**：成诗态会收起待选字和入场文案，只复用宣纸；完整短诗进入纸面两行排版，选中字作为“种子字”显示在左侧轴首。
- **诗库不做随机生成**：穷举 56 个组合比模板拼接更稳，便于审校，也避免 AI 文案不可控。
- **五言二句作为统一体裁**：宣纸横幅内两行五言最稳，既有古典短章感，又不会像律诗一样过度正式。
- **解释层能删则删**：来源说明、“这些字从哪里来”“屈原 · 香草 · 楚江”和“字从水起”这类独立提示若不能增强画面，应删除或融入诗句本身。
- **非游戏化表达保持不变**：不出现分数、倒计时、正确、错误、失败、通关或评级。
- **竹影叠层撤下**：`bg-bamboo.webp` 有矩形底，在新人物背景上会形成右侧大方块；第五幕不再叠加该素材。
- **水痕承接为视觉模拟**：由于场景切换会销毁第四幕 Pixi 容器，第五幕暂不直接复用第四幕对象，而是在第五幕入场重新绘制水痕和承接字，保持叙事连续。

## 诗库质量准则

当前诗库采用“稳定诗库 + 意象分组 + 五言二句”。

- `江 / 月 / 风`：负责水气、时间、流动和画面空间。
- `兰 / 艾`：负责香草、楚辞、端午清气。
- `舟 / 鼓`：负责竞渡行动和第四幕承接。
- `钟`：负责礼乐回声和第六幕过渡。

写作时不要机械把 3 个字全部硬塞进诗句，而是让选字决定意象场。允许诗句显性回应 2 个字，第三个字通过语义或气质进入。需要避免“字粒”“问短诗”“起回声”这类程序说明词，也避免口号式、百科式、现代白话式表达。

代表方向：

- 江月浮清影，微风入楚歌。
- 兰影沉江月，清光照楚辞。
- 鼓急催兰舟，风开楚水纹。
- 艾束悬舟首，鼓声过晚江。
- 鼓歇江风定，钟声入暮烟。
- 兰艾含清露，江风送远香。
