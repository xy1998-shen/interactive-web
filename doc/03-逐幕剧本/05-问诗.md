# 05 问诗

## 场景描述

上一幕结尾，龙舟驶过后留下长水痕，水痕中浮出“江”“艾”“舟”“鼓”“风”等字粒。用户点击“入下一幕”后，水痕扩散为深青静水，朱砂余韵退成淡金和雾白，江面速度明显放慢。

画面采用宽幅楚江集市背景缓慢横移：左侧歌舞与舟上人物先入画，中段江亭、远山与柳枝展开，最终定格在临水诗人和远钟将起的江面。整体速度要慢，留出“从竞渡动势落回静水问诗”的呼吸感。

画面移动结束后，中央偏下展开一张横向浅色宣纸诗笺。8 个中文意象字写在纸面上：江、艾、舟、鼓、风、月、兰、钟。用户不是在水面“点散字”，而是在诗笺上以毛笔圈字问诗；字与纸、圈痕和诗句结果必须处在同一视觉系统内。

## 目的

- 承接第四幕”鼓声已渡”和水痕字粒，把竞渡动感落回静水。
- 让用户从”众人合力”回到”个人求索”，通过触字问出一行短诗。
- 用诗字散成声波，自然引出第六幕”和鸣”。

## 交互

1. 入场时，宽幅江景从左至右缓慢平移，画面移动期间不提前显示底部操作提示。
2. 背景定格后，底部动作提示显示“轻触三字，问一行诗”，中央宣纸诗笺淡入。
3. 8 个字写在宣纸中央留白区，两行排布，不做现代按钮、卡片或棋盘格。
4. 用户每轮选择 3 个字。未满 3 个时不生成半成品诗句。
5. 用户触字时，字略放大、提亮，并出现不规则毛笔圈痕；毛笔圈痕比水纹更重要，水纹只作为弱反馈。
6. 用户在未满 3 个字时再次点击已选字，可取消选择；字、圈痕和位置都回到基础状态。
7. 第 3 个字选中后，系统完成本轮问诗。未选字淡出，3 枚选中字转为宣纸左侧轴首小字，字号要足够明显。
8. 原创短诗直接在同一张宣纸上生成，不另起诗歌区域，不与待选区域重叠。
9. 成诗后在宣纸右下角显示“重新圈字”。用户点击后，当前短诗淡成水痕回声，8 个字重新回到纸面，可继续组诗。
10. 每轮只突出当前 1 条短诗，不把多轮结果累积成诗墙。
11. 不再显示“这些字从哪里来”“屈原 · 端午 · 楚江”等解释性标签；如文案不能增强叙事，应直接删去。
12. 成诗态不显示额外完成提示，例如“字从水起”。宣纸里的诗句本身就是完成反馈。
13. 用户点击「入下一幕 →」后，当前诗字拆成笔画光点，沿水面扩散成圆形声波，远处浮现青铜钟影，进入“和鸣”。

播放过程中不显示分数、倒计时、正确、错误、失败、通关或评级。第五幕不是答题游戏，用户是在水面问诗，不是在选择标准答案。

## 技术

- 字粒使用 Pixi Text 或 DOM 文本，保持清晰可读，点击区域应略大于视觉文字。
- 字粒配置允许为空；入口函数应做防御检查，避免空数组取模或 `undefined` 文案进入渲染。
- 字粒进入场景时记录 `baseX`、`baseY`、`baseAlpha`、`baseScaleX/Y`。漂浮、选中、取消、再问和完成态都基于 base 值计算。
- 同一字粒的新动画开始前先 `killTweensOf` 字粒本体和 `scale`，避免多轮组诗后位置或缩放漂移。
- 每轮成诗需要最少 3 个字、最多 3 个字。第 3 个字后完成本轮，不追加第 4 个字。
- 匹配逻辑必须确定性：同一组选字永远返回同一首诗。
- 生成 key 时将 3 字按固定字典序排列。
- `POEM_DB` 必须穷举 8 个字任取 3 个的全部 56 种组合；正常路径不依赖 2 字降级匹配。
- 兜底句只用于异常输入或后续字表扩展时的防御，不作为常规体验的一部分。
- 禁止接入在线随机生成，避免不可控文案。
- 每轮成诗由单一函数接管：生成短诗、绘制选中字轴首、显示“重新圈字”和显示下一幕入口都在同一状态边界内完成。
- 成诗呈现复用宣纸诗笺，不另铺弹窗、诗歌卡片或第二结果区。3 枚选中字作为诗句种子进入纸面左侧，短诗在纸面中部按两行清楚排版。
- 成诗态文字使用 `ChuJiangCalligraphy` 或 `STKaiti, Kaiti SC, KaiTi, Songti SC, STSong, Noto Serif CJK SC, serif` 书法字体栈，不使用 `ChuJiangWenKai`，避免轻松儿童感。
- 宣纸背景使用透明 PNG 素材：边缘自然不规则，带手工纤维、淡墨水痕、极淡水波纹和克制青铜金线饰；不使用卷轴杆、厚重边框、卡片阴影或现代 UI 装饰。
- 快速重新圈字、快速切场和恢复完成态时，短诗、纸笺、重圈入口和入口按钮都要做空值检查。
- 诗字散成声波可用 Pixi Graphics 圆环 或 Pixi Graphics 圆环实现，扩散时长建议 1200ms 至 1800ms。

## 文案

- 入场短句：鼓声入水，字从痕起。
- 动作提示：轻触三字，问一行诗
- 再问题签：重新圈字
- 字粒：江、艾、舟、鼓、风、月、兰、钟
- 完成短句：兰艾入水，风过楚江。
- 过渡短句：诗声入殿，远钟将鸣。

### 短诗库方向

诗库穷举 56 个三字组合，统一为“五言二句”：`五字，五字。`。短句均为原创，不直接引用长篇经典文本。每轮只突出当前 1 条短诗，上一轮淡成水痕回声或淡出。

写作原则：

1. 不机械把 3 个字全部硬塞进诗句，而是让选字决定意象场。
2. `江 / 月 / 风` 负责水气、时间和流动。
3. `兰 / 艾` 负责香草、楚辞和端午清气。
4. `舟 / 鼓` 负责竞渡行动和第四幕承接。
5. `钟` 负责礼乐回声和第六幕过渡。
6. 避免“字粒”“问短诗”“起回声”这类偏程序说明的表达。
7. 诗句风格保持低饱和、含蓄、清润，不写口号，不写百科解释。

代表句：

- 江月浮清影，微风入楚歌。
- 兰影沉江月，清光照楚辞。
- 鼓急催兰舟，风开楚水纹。
- 艾束悬舟首，鼓声过晚江。
- 鼓歇江风定，钟声入暮烟。
- 兰艾含清露，江风送远香。

完整诗库见 `js/scenes/poem-scene.js` 中 `POEM_DB`；异常兜底见 `FALLBACK_POEMS`。

### 成诗呈现

成诗不是普通结果文本。第 3 个字选中后，三枚选中字先转为宣纸左侧轴首，完整短诗再进入同一张纸：

1. 3 枚选中字不再停留在待选字阵中，改为纸笺左侧青铜金竖排小字，字号和描边必须足够清楚。
2. 完整诗句按两行进入纸笺中部，优先保证清晰、稳定、不重叠。
3. 宣纸是诗句承载物，不再另铺水墨绢纸、结果卡片或右侧诗歌区域。
4. “重新圈字”放在宣纸右下角，作为轻题签，不放画面左侧或底部中心。
5. 同屏始终只保留当前短诗，避免形成诗墙。
6. 成诗态不同时显示待选字阵、入场短句、解释性知识文案和完成提示，避免任何文字层互相压住。
7. 短诗和三字轴首使用 `ChuJiangCalligraphy` / 楷书书法 fallback，不能回退到 `ChuJiangWenKai` 作为首选字体。

## 验收要点

- 入场能明确承接第四幕：水痕扩散、字粒入诗，而不是独立选字页。
- 人物只能作为远岸题诗或咏诗的叙事来源，不能过大、不能居中、不能遮挡中央和偏右的水面留白。
- 背景里的竹简、素笺和水面不能出现可读文字，避免和前景字粒、短诗、诗轴冲突。
- 8 个字必须清楚可读，漂浮和水纹不影响辨认。
- 字写在宣纸上，点击反馈像毛笔圈字，不像答题按钮、棋子或卡牌。
- 每轮最少 / 最多 3 字成诗；未满 3 字不生成短诗，满 3 字后不追加第 4 字。
- 取消选择和“重新圈字”都温和，不出现错误、失败、扣分或倒计时。
- 同一组选字生成同一条短诗；8 个字任取 3 个的 56 种组合都必须直接命中诗库，不接入在线随机生成。
- 每轮短诗进入同一张宣纸，不做普通居中结果文本、右侧结果区或诗墙。
- 不显示解释性关键词和轻知识文案；如后续需要文化补充，应融入诗句或过渡，而不是另开说明层。
- 完成态能停留并支持继续组诗：当前短诗、选中字轴首、“重新圈字”和入口都可读。
- 点击入口后，诗字能自然散成声波，引出远处青铜钟影和第六幕“和鸣”。
- 成诗后短诗以宣纸形式呈现（同一纸笺 + 三字轴首 + 两行排版），不是弹窗、卡片或普通居中文本。
- 纸笺、圈痕和字体不显儿童感；缺少 Aa 字体时仍用系统楷体 / 宋体 fallback 保持清晰。
- 本地静态服务打开时背景、字粒、短诗和交互都正常显示，不依赖远程资源。
