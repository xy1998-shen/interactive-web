# 06 和鸣

## 场景描述

第五幕结尾，诗字散成声波，远处钟影将鸣。进入第六幕时，画面不再继续展开江面，而是转入一处被雾光照亮的楚地礼乐空间：木梁、竹帘、帷幔、青铜纹样和石台构成半室内乐台，远处乐人正侧身击钟。

这张背景的作用是把作品从"水上问诗"转到"礼乐合鸣"。画面左侧或中远景保留低对比编钟和乐人，证明礼乐正在发生；中央和右侧是明亮雾光、帷幔、浅色墙面与石台留白，用于承载三处交互回声节点、青铜声波、诗字碎片和前文风物闪回。

前景不再放一个巨大的可点击编钟。编钟只作为小型声波锚点或视觉提示出现，真正的叙事目标是场景里一条低透明青铜声路上的三处"回声节点"：诗声、风物、礼乐。用户不需要完成三个孤立任务点；进入本幕后，系统顺着诗字入殿后的声波路径自动触发三处回声，让第五幕诗字、前几幕物象和编钟礼乐逐层合在一起，最终收束成尾声端午印记的前奏。

### 入场：诗声入殿

上一幕的诗字光点沿水面散开后，不再进入开阔江景，而是像被风带入帷幔。画面切到明亮雾白的楚乐台：竹帘轻垂，青铜纹样在墙面和石台上若隐若现，远处乐人和编钟处在左侧雾光里。

过渡动画：第五幕水面从下方渐隐，帷幔和竹帘从画面上方帘幕式推入；诗字光点穿过帷幔缝隙进入室内，变为墙面和石台上的淡金波纹。少量水纹反光只保留在石台边缘，作为"诗声入水"到"诗声入殿"的视觉桥接，但不形成水面主体。整体时长建议 1.2s 至 1.8s。

动作提示：

> 听三处回声次第和鸣

入场短句：

> 诗声入殿，远钟将鸣。

### 第一处回声：诗声

第一个回声节点位于中央偏左的雾光或石台边缘，标签为"诗声"。节点下方可低透明显示短注释"字落成声"，只作为当前回声的场景标注，不做按钮说明。自动演出到这一段时，青铜金声波从节点处扩散，几枚第五幕字粒沿声波边缘浮起：江、艾、舟、鼓、风、月、兰、钟。

它们不是新一轮问诗，也不组成长句，只像上一幕短诗留下的回声。诗字要比普通氛围粒子更清楚：透明度可提升到 35% 至 40%，扩散结束后至少停留 1 秒以上，再慢慢淡入帷幔和雾光。

短句：

> 字落成声。

### 第二处回声：风物

第二个回声节点位于画面中右或浅色墙面前，标签为"风物"。节点下方可低透明显示"艾粽舟影"。自动演出到这一段时，前几幕物象以低透明度短暂闪回：艾草、粽叶、粽形、龙舟水痕或舟影。

这些元素不要围成装饰墙，也不要压住背景乐人。它们应该像被钟声照见的记忆片段，在乐台空间里出现一瞬，然后退回纹样和雾气。

短句：

> 艾、粽、舟、诗，一声相和。

注：此处用"诗"是回忆第五幕问诗意象；过渡处用"钟"是引出尾声编钟印记节点，两者语境不同。

### 第三处回声：礼乐

第三个回声节点位于靠近小编钟或声波中心的位置，标签为"礼乐"。节点下方可低透明显示"楚江和鸣"。自动演出到这一段时，前景小钟影微振，三层谐波编钟音同时响起；远处乐人作为静态礼乐背景保留，不额外叠加局部动作。

这一声不再只是向外扩散，而是让前两处声波慢慢回流。诗字碎片、风物残影和青铜纹样共同向中心收束。

完成短句：

> 金声入水，楚江和鸣。

### 完成态：三声相和

完成态不自动跳走。画面停留在明亮楚乐台中，三处已点亮节点保持低亮，右侧或中部信息层显示"三声相和"的解释：

- 诗字随声波散开，成为第一层回声。
- 艾、粽、舟的记忆短暂浮现，回应前路。
- 编钟礼乐与江声合拢，收束为端午印记。

声波边缘出现轻知识光点。用户点击后展开轻知识；不点击也可以继续进入下一幕。

轻知识：

> 编钟礼乐承载荆楚记忆；在这里，它与江声、诗字和端午风物一起合鸣。

### 过渡：声波收束成印记

用户点击「入下一幕 →」后，三处回声节点的光沿青铜声波向中心汇合。中心短暂出现"艾、粽、舟、钟"四个小光点，进入第七幕"端午印记"。

过渡原则：

- 不硬切。
- 不回到大江面。
- 不把第六幕做成终章；它只是让礼乐空间中的三处回声合为尾声印记。

## 目的

- 避免第六幕继续重复第一幕大江面、第五幕静水问诗的构图。
- 用"楚乐台 / 礼乐厅"建立新的空间记忆，让第六幕承担文化升华而不是又一次水面交互。
- 把前文诗字、艾粽舟物象和编钟礼乐合为一次可操作的三段回声。
- 为第七幕"端午印记"提供清晰过渡：三处回声收束为"艾、粽、舟、钟"四个印记点。

## 交互

1. 入场过渡：第五幕水面渐隐，帷幔推入，诗字光点穿入楚乐台变为淡金波纹。显示楚乐台背景、远景乐人和低对比编钟。
2. 场景内出现一条低透明青铜声路，声路上有三个回声节点：诗声、风物、礼乐。节点只作为视觉标记，不作为点击热区；无错误提示、扣分或失败。
3. 自动触发"诗声"：诗字碎片与第一层声波浮现，显示"字落成声。"
4. 自动触发"风物"：艾草、粽叶、龙舟等前文物象低透明闪回，显示"艾、粽、舟、诗，一声相和。"
5. 自动触发"礼乐"：编钟音、回流声波和完成短句"金声入水，楚江和鸣。"
6. 完成态显示"三声相和"信息层和轻知识光点。光点点击展开轻知识，停留 3 至 5 秒后淡出，光点保留可再次查看；不点击不阻塞推进。
7. 完成态出现「入下一幕 →」入口；点击后三处回声收束为尾声印记前奏。

界面不显示"回声 1/3"这类计数文案。节点本身的亮/暗状态已经传达了进度。

## 技术

- 第六幕主背景使用 `assets/backgrounds/bg-bell-water.png`，它是楚乐台 / 礼乐厅空间，不再是江面背景。
- 前景编钟缩小为声波锚点，宽度建议不超过视口 8% 至 10%；不能遮挡背景乐人和中部留白。
- 三处回声节点用 Pixi Graphics + Text 生成，节点标签为"诗声 / 风物 / 礼乐"，短注释为"字落成声 / 艾粽舟影 / 楚江和鸣"。
- 节点之间增加低透明青铜声路：常态只作为空间线索；每段自动触发后点亮已走过段落；第三声完成时连到小编钟锚点，作为进入尾声的回流前奏。
- 节点严格按顺序自动点亮：节点保持 `eventMode: none`，不作为点击热区。无需错误提示逻辑。
- 节点点亮后停止呼吸，透明度压低，避免完成态过亮。
- 声波用 Pixi Graphics 多层圆环实现，圆心绑定当前节点，而不是固定鼠标点或巨大编钟中心。
- 诗字碎片和"艾 / 粽 / 舟 / 钟"收束字都用 Pixi Text 生成，不切图。
- 风物闪回复用现有艾草、粽叶、粽子、龙舟素材，透明度控制在 10% 至 18%。
- 低透明端午纹样可叠在楚乐台中部或石台区域，增强礼乐空间质感，但透明度需克制，不能形成一张独立装饰贴图。
- 触发音使用 Web Audio API 三谐波递进：第一声单谐波，第二声双谐波，第三声三谐波并延长衰减。
- 自动演出需有触发间隔，间隔应覆盖诗字扩散、短暂停留、主要声波和金尘动画，建议 4s 左右。
- 完成态轻知识通过光点展开，不点击不阻塞进入下一幕。
- 第六幕保留左上章节标题，与前几幕统一；标题层需要保持轻量，不遮挡背景乐人和编钟。信息改放场景内右侧或中部留白区域。
- 背景叠层应比第五幕更明亮，避免再形成深青水面观感。
- 入场过渡动画：水面从下方 alpha 渐隐 → 帷幔从上方推入（translateY 或 mask reveal）→ 诗字光点穿入变为淡金波纹。建议总时长 1.2s 至 1.8s。

字体要求：入场短句、完成短句和节点标签使用 `ChuJiangCalligraphy` 或楷书 fallback（`STKaiti, Kaiti SC, KaiTi, Songti SC, STSong, Noto Serif CJK SC, serif`），与第四幕、第五幕一致。轻知识和信息层文本使用 `ChuJiangWenKai` 正文字体栈。

## 验收要点

- 第六幕必须明显区别于第一幕大江面和第五幕问诗静水。
- 入场过渡清晰：水面渐隐 → 帷幔推入 → 光点变为室内波纹，不能硬切。
- 背景应是楚乐台 / 礼乐厅空间，水只作为极少量反光或过渡线索。
- 左上角不应出现章节标题、背景乐人和说明文字互相压住的问题。
- 前景编钟不能巨大居中；它只是声波锚点。
- 交互不是"反复点编钟"，也不是三个孤立任务点；三处回声沿一条声路自动按顺序点亮。
- 节点不作为点击热区；无错误、扣分或失败。
- 三处节点能分别触发诗字、风物、礼乐三段反馈。
- 第一段诗字回声必须清楚可读，不能像低透明粒子一闪而过。
- 信息量比旧版更清楚：完成态应能读到"三声相和"的层次说明。
- 完成态轻知识光点可点击展开，不点击不阻塞推进。
- 触钟时有编钟合成音反馈，三次触发音色递进；关闭声音后视觉体验完整。
- 背景整体更明亮，不出现大面积暗青黑。
- 点击入口后，三处回声能自然收束成尾声印记前奏。
- 本地静态服务打开时不依赖远程素材、远程字体、模块加载或联网接口。

## 优化沉淀

第六幕已经从“点击式节点交互”调整为“自动声路演出”。后续维护时不要把它重新做成新的任务关卡，它承担的是第五幕“问诗”到第七幕“端午印记”的过渡。

### 核心决策

1. 承接第五幕诗字散成声波。
2. 召回前几幕艾、粽、舟等端午物象。
3. 以编钟礼乐把声波收束到第七幕印记。

当前不需要重新生成 `bg-bell-water.png`。现有背景已经具备楚乐台、竹帘、帷幔、远景乐人和低对比编钟，右侧留白也足够承载“三声相和”信息层。主要维护点是运行时叠层和文字关系：声路、节点、提示文案和完成态需要克制地叠进背景。

### 当前节奏参数

关键参数集中在 `js/config.js` 的 `bell` 配置中：

- `entryDuration: 1.65`
- `autoStartDelay: 0.5`
- `autoStepDelay: 4.2`
- `poemEchoAlpha: 0.38`
- `poemEchoFontSize: 30`
- `poemEchoSpreadDuration: 1.65`
- `poemEchoHoldDuration: 1.35`
- `poemEchoFadeDuration: 0.9`

诗字可读性优先级高于速度。后续调节时，不能把诗字重新压回低透明粒子，也不能让第一段诗声在 1 秒内消失。

### 文字与遮挡规则

- 保留左上 `06 / 楚江寻艾` 和大标题 `和鸣`。
- 不显示左侧副文案。
- 动作提示使用底部题签。
- 完成态不显示 DOM 完成短句 `钟已鸣`，避免与 Pixi 节点 `礼乐 / 楚江和鸣` 重叠。
- 完成态说明放在右侧留白区域，标题为 `三声相和`。

如果后续要加文案，应优先放到右侧留白或底部题签，不要放在左侧乐人、中央声路和礼乐节点附近。

### 实现入口

运行时代码集中在 `js/scenes/bell-scene.js`：

- `playBellEntryTransition`：入场水面渐隐、帷幔推入、诗字光点入殿。
- `startBellAutoSequence`：入场结束后启动自动三段回声。
- `tapBell`：保留为内部触发函数，由自动时间线调用，不再由点击调用。
- `createBellSoundPath` / `redrawBellSoundPath`：绘制常态声路和已点亮声路。
- `showBellPoemEcho`：诗声字粒扩散、停留与淡出。
- `showBellFlashback`：风物闪回。
- `completeBell`：完成态信息层和下一幕入口。
- `updateBellLayout`：窗口尺寸变化时重排纹样、声路、节点、编钟、信息层和知识光点。

不要重新引入节点 `pointertap` 作为主流程入口。轻知识光点仍可点击，但它不阻塞进入下一幕。

### 不要回退的方向

- 不要重新做成“点击三个节点”的任务链。
- 不要把前景编钟放大成点击主体。
- 不要为了补信息在左侧人物区域叠长文案。
- 不要把诗字回声降成低透明装饰粒子。
- 不要为第六幕单独生成一套和第七幕不连续的新背景风格。
