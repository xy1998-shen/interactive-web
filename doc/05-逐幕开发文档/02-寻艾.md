# 02 寻艾

## 场景描述

左侧是一条延续入江场景的雾青江水和薄雾水路；右侧是清晨江边村庄，有屋舍、窗户、门楣和青石村路。村路上有多位背着艾草的老人，形成端午清晨“采艾归村”的生活感。

画面最近的一户人家窗前是主视觉焦点：有人正抬手准备把艾草枝挂到窗前或门楣处。用户点击画面任意位置后，艾草沿香气轨迹自动飞向窗前或门楣，并触发人物完成挂艾动画。

## 目的

- 把“寻艾”从江面找物改成“轻点画面、送香入户”的自动演出动作。
- 讲清楚端午悬艾的民俗来源：仲夏草木、避疫祈安、悬艾守户。
- 保留与端午追思的连接，但不把悬艾全部归因于屈原。

## 交互

- 用户点击画面任意位置（全屏均可触发，不要求精确点中艾草），进入自动悬艾演出。
- 艾草沿预设曲线飞向最近屋舍的窗前或门楣，过程留下淡绿色香气轨迹。
- 目标区域出现柔和吸附光，艾草到位后吸附到门楣或窗前。
- 窗前人物抬手接过艾草，完成挂艾动作。
- 挂艾完成后显示短句”艾气入户，清风守门”，画面出现「下一幕」入口提示，用户主动点击进入”裹青”，不自动硬切。

## 技术

- 场景透明触发层覆盖全画面，监听 `pointertap` 后启动自动时间轴。无拖拽逻辑。
- 艾草精灵保留为视觉焦点，不要求用户精确点中艾草。
- 自动飞行过程使用粒子池生成艾草青轨迹，粒子 600ms 至 900ms 淡出。
- 目标区域使用窗前和门楣矩形或圆形视觉提示，纯视觉反馈，不承担命中判定。
- 艾草到达目标点后自动吸附并锁定位置，播放人物挂艾序列动画。
- 人物动画可先用剪影分层和 GSAP 位移/旋转实现，后续再替换为精细帧动画。
- 完成态将香气轨迹转为粽叶折线（青色折线），作为下一幕"裹青"粽叶展开的视觉起点。

## 文案

- 短句：艾气入户，清风守门。
- 轻知识：端午悬艾，寄托避疫、祈安与迎夏的愿望。
- 画轴方向：五月草木正盛，采艾归村，悬艾守户；再用一句连接端午追思。

### 画轴文案草案

1. 五月草木正盛，艾叶带着清苦香气，从江岸走进村巷。
2. 古人以艾为芳草，《荆楚岁时记》载端午采艾悬户，以禳毒气、护盛夏。
3. 老人背艾归来，家家门窗添一束清芬，愿疾病远避，家宅平安。
4. 后来端午兼有追思屈原之意，艾香、粽叶与江上竞渡，便一同成为五月的记忆。

> 完整文案详见 `doc/01-项目原则/页面构思.md`。

## 验收要点

- 一眼能看出”左江右村”的空间关系（左侧江面约 35%-45% 画面宽度）。
- 老人背艾归村是生活背景，不能抢过窗前挂艾主焦点。
- 全屏任意位置点击均可触发完整自动悬艾演出，不能只闪一下或直接跳完成态。
- 无拖拽交互，用户不需要精确点中艾草或拖动它。
- 自动飞行时有淡绿色香气轨迹，靠近目标时有吸附光反馈。
- 完成态能明确看见艾草挂在窗前或门楣。
- 演出结束后出现「下一幕」入口提示，用户主动点击才推进。
- 完成后香气轨迹转为青色折线，自然引出下一幕”裹青”粽叶展开。
- 本地 `file://` 打开时背景、村庄、艾草精灵和文字都正常显示。
