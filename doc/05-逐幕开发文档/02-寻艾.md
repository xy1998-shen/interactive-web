# 02 寻艾

## 场景描述

第二幕当前采用 4 个生活场景顺序播放，围绕“艾草如何进入端午日常”展开：悬艾、煮汤、制糕、佩香。每个子场景使用独立背景和少量 Pixi 动效，整体仍保持雾青、米白、草木青的水墨空间。

画面左上保留章节标题层：`02 / 楚江寻艾` 与标题字 `寻艾`。第二幕和第三幕标题层保持一致，不额外显示副标题说明，避免遮挡背景主体。

## 目的

- 把“寻艾”从单次找物改成一组连续生活片段，呈现悬艾、艾汤、青团、香囊等端午草木日常。
- 讲清楚端午悬艾的民俗来源：仲夏草木、避疫祈安、悬艾守户。
- 保留与端午追思的连接，但不把悬艾全部归因于屈原。
- 与第三幕共用同一套标题层、面板说明、底部圆点和右下知识卡规则，形成统一体验。

## 交互

- 第二幕进入后自动按顺序播放 4 个面板：悬艾 → 煮汤 → 制糕 → 佩香。
- 自动播放只走一遍，到第 4 个面板后停住，不循环回第 1 个面板。
- 用户可以点击底部圆点手动切换面板；圆点只是面板导航，不再构成第二套剧情任务。
- 四个面板都到达后，右下知识卡自动展开，随后出现「入下一幕 →」入口。
- 知识卡无需点击触发，默认展开，不提供收起态。

## 技术

- 使用 `initPanels()` 创建 4 个等尺寸面板，面板背景分别为 `mugwortPanel1Hang`、`mugwortPanel2Boil`、`mugwortPanel3Cake`、`mugwortPanel4Sachet`。
- 使用 `startAutoPlay()` 顺序播放。公共 `PanelSwitcher` 播放到最后一张后自动停止，不回到首面。
- 悬艾面板使用艾草束素材轻摆和香气线；煮汤面板使用蒸汽素材与少量蒸汽粒子；制糕面板使用粉雾素材；佩香面板使用香囊清气素材。
- 右下知识卡通过 `DOMLayer.showKnowledgeCard()` 自动出现，标题为“端午寻艾 · 悬艾守户”。
- `hero-copy` 不再对第二幕加 `is-suppressed`，保证左上章节标题和第三幕对齐。

## 文案

- 面板标题：悬艾、煮汤、制糕、佩香。
- 知识卡标题：端午寻艾 · 悬艾守户。
- 知识卡方向：解释为何端午悬艾、艾草如何进入日常、悬艾与端午记忆的关系。
- 章节标题层：只显示 `02 / 楚江寻艾` 与 `寻艾`，不显示副标题说明。

### 知识卡文案要点

1. 农历五月草木正盛，艾叶气味清苦，古人悬在门窗旁，寄托避疫、祈安与迎夏。
2. 艾草进入日常的方式包括悬门、煮汤、入食、佩香。
3. 悬艾不只来自屈原追思，也来自盛夏生活经验；后来它与粽叶、竞渡一起成为端午共同记号。
