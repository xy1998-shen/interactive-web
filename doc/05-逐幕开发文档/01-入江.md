# 01 入江

## 场景描述

清晨或暮色中的楚江水面，整体是明亮雾青色。江面有低低流动的薄雾，远处有一叶很小的水墨舟影，右上保留柔和暖金天光。画面中下部有淡淡水纹和微弱光点，暗示用户可以轻点江面开启旅程。

左侧画轴在交互后展开，以诗意短文引出楚江、屈原、端午追思，以及艾、粽、竞渡这些后续线索。

## 目的

- 建立《楚江寻艾》的第一眼气质：雾青江面、远舟、晨光、端午旧事。
- 让用户通过一个轻动作进入旅程，而不是点击普通开始按钮。
- 为后续“寻艾”保留水路延续和江上行舟的叙事方向。

## 交互

- 用户轻点江面（画面中下部水纹区域为主 hit area，但全屏点击均可触发）。
- 点击后进入自动演出：”入江寻艾”四字在画面中浮现再淡出。
- 雾气从左下约 30° 向右上约 30° 斜向散开，小舟沿水路缓慢向朝阳处行进。
- 画轴展开并逐句显示入江叙事，用户不需要持续拖动。
- 演出结束后画面出现「下一幕」入口提示（文字按钮或光点），用户主动点击进入”寻艾”，不自动硬切。
- 退场过渡：水纹向岸边流去，带出艾草轮廓，衔接下一幕”左江右村”空间。

## 技术

- 使用 Pixi 背景层、雾气层、水纹层和 DOM 文字层。
- 雾气遮罩可使用 RenderTexture：点击后按时间轴在遮罩上绘制透明区域。
- 雾气层叠加噪声置换，让雾流动更自然。
- 水纹点击反馈使用圆环 sprite / SVG / Graphics 动态扩散。
- 场景完成条件：入江动画完成且画轴文案播放结束。

## 文案

- 主标题：楚江寻艾
- 副标题：一场屈原故里端午的沉浸式交互诗
- 动作提示：拨开江雾
- 画轴方向：楚江晨雾、屈原行吟、汨罗追思、后人以艾粽竞渡寄托怀想。

### 画轴文案草案

1. 楚江晨雾未散，舟行旧水之间。两千余年以前，三闾大夫屈原曾在楚地行吟，把家国之忧写进江风。
2. 《离骚》有言："路漫漫其修远兮，吾将上下而求索。"那一声求索，后来随五月江水，被一代代人记起。
3. 相传屈原沉汨罗后，乡人竞舟追寻，以角黍投江，愿鱼龙不伤其身；又悬艾佩兰，以草木清气护住盛夏门户。
4. 于是端午不只是一日节令，也是后人把忠贞、忧思与祝愿，一年年送回江上的方式。

> 实际实现时可根据节奏删减为 2-3 句，避免遮挡太阳和小舟。完整文案详见 `doc/01-项目原则/页面构思.md`。

## 验收要点

- 首屏不是黑夜图，必须有明亮雾青江光。
- 小舟尺寸克制（画面宽度 8%-12%），不能抢标题和水纹入口。
- 雾气散开方向为斜向（左下→右上约 30°），不是机械横扫。
- 轻点后自动播放入江演出，画轴文案完整播放，用户可阅读后再推进。
- 演出结束后出现「下一幕」入口提示，用户主动点击才推进，不自动硬切。
- 退场时水纹向岸边流动并带出艾草轮廓，自然引入寻艾。
- 本地 `file://` 打开时背景、雾气、小舟和文字都正常显示。
