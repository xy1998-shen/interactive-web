# 05 问诗

## 场景描述

上一幕结尾，龙舟驶过后留下长水痕，水痕中浮出“江”“艾”“舟”“鼓”“风”等字粒。用户点击“入下一幕”后，水痕扩散为深青静水，朱砂余韵退成淡金和雾白，江面速度明显放慢。

画面是一片安静的深青水面，远处保留低对比远山、薄雾、竹影或竹简暗影。左侧远岸可出现一位中远景古代诗人，临水题诗或咏诗；石案上的竹简或素笺只保留淡墨痕，不出现可读文字。诗人笔尖的墨意落入水面，化成极淡涟漪和未成形字点，成为前景字粒浮起的叙事来源。

水面逐步浮现 8 个中文意象字粒：江、艾、舟、鼓、风、月、青、钟。字粒像从水痕和墨意中被问起，不排成答题选项，也不做按钮网格。人物只提供”题诗入水”的氛围和来源，不遮挡中央与偏右水面留白。

## 目的

- 承接第四幕”鼓声已渡”和水痕字粒，把竞渡动感落回静水。
- 让用户从”众人合力”回到”个人求索”，通过触字问出一行短诗。
- 用诗字散成声波，自然引出第六幕”和鸣”。

## 交互

1. 入场时，第四幕水痕扩散成深青静水，显示短句“鼓声入水，字从痕起。”。
2. 水痕稳定后，8 个字粒从水下轻浮出现：江、艾、舟、鼓、风、月、青、钟。
3. 底部动作提示显示”轻触三字，问一行诗”。
4. 用户每轮选择 3 个字粒。未满 3 个时不生成半成品诗句。
5. 用户触字时，已选字轻亮、上浮并生成水纹。周边水面可产生轻反馈（如艾叶微漂、舟痕扩散、月影泛光或钟纹轻亮），此为可选增强，MVP 阶段统一用水纹即可。
6. 用户在未满 3 个字时再次点击已选字，可取消选择；字粒轻落回水面。
7. 第 3 个字选中后，系统完成本轮问诗。入场短句退场，8 个字粒全部退场（包括 5 个未选字淡出），只在水面中部展开一条淡淡横向诗轴。
8. 原创短诗进入诗轴中，按两行清楚排版；3 枚选中字作为轴首小字点亮，提示诗句从用户触过的字里生长出来。
9. 成诗后显示”再问一诗”轻题签。用户点击后，当前短诗拆成小水点或淡成水痕回声，8 个字粒重新浮现于水面，可继续组诗。
10. 每轮只突出当前 1 条短诗，不把多轮结果累积成诗墙。
11. 短诗下方或旁侧出现关键词“屈原 · 端午 · 楚江”，用户悬停或点击后补全轻知识。
12. 用户点击「入下一幕 →」后，当前诗字拆成笔画光点，沿水面扩散成圆形声波，远处浮现青铜钟影，进入“和鸣”。

播放过程中不显示分数、倒计时、正确、错误、失败、通关或评级。第五幕不是答题游戏，用户是在水面问诗，不是在选择标准答案。

## 技术

- 字粒使用 Pixi Text 或 DOM 文本，保持清晰可读，点击区域应略大于视觉文字。
- 字粒配置允许为空；入口函数应做防御检查，避免空数组取模或 `undefined` 文案进入渲染。
- 字粒进入场景时记录 `baseX`、`baseY`、`baseAlpha`、`baseScaleX/Y`。漂浮、选中、取消、再问和完成态都基于 base 值计算。
- 同一字粒的新动画开始前先 `killTweensOf` 字粒本体和 `scale`，避免多轮组诗后位置或缩放漂移。
- 每轮成诗需要最少 3 个字、最多 3 个字。第 3 个字后完成本轮，不追加第 4 个字。
- 匹配逻辑必须确定性：同一组选字永远返回同一首诗。
- 生成 key 时将 3 字按固定字典序排列。
- 精确命中用对应诗句；未命中则找包含 2 字的首条（降级匹配按预排序 key 列表扫描，保证所有引擎结果一致）；仍未命中使用第一个选中字的兜底句。
- 禁止接入在线随机生成，避免不可控文案。
- 每轮成诗由单一函数接管：生成短诗、绘制三字成脉诗轴、显示关键词、显示“再问一诗”和显示下一幕入口都在同一状态边界内完成。
- 成诗呈现不使用普通居中结果文本。3 枚选中字应作为诗句种子进入诗轴，短诗在半透明横向诗轴中按两行清楚排版，保持水面生成感。
- 快速再问、快速切场和恢复完成态时，短诗、关键词、再问题签、入口按钮和轻知识对象都要做空值检查。
- 诗字散成声波可用 `assets/svg/vec-ripple-ring.svg` 或 Pixi Graphics 圆环实现，扩散时长建议 1200ms 至 1800ms。

## 文案

- 入场短句：鼓声入水，字从痕起。
- 动作提示：轻触三字，问一行诗
- 再问题签：再问一诗
- 字粒：江、艾、舟、鼓、风、月、青、钟
- 完成短句：字从水起，风过楚江。
- 关键词：屈原 · 端午 · 楚江
- 轻知识：屈原诗意与端午记忆相连，是楚江旅程的精神底色。
- 过渡短句：诗声入水，远钟将鸣。

### 短诗库方向

- 江风拂艾，五月入舟。
- 青叶藏愿，鼓声过江。
- 舟影穿雾，楚水生光。
- 艾香近岸，钟波向远。
- 月落江心，字起成诗。
- 鼓歇舟远，青波留声。
- 风过艾叶，江月生凉。
- 钟影未至，诗已入水。

短句均为原创，不直接引用长篇经典文本。每轮只突出当前 1 条短诗，上一轮淡成水痕回声或淡出。以上为代表方向，完整诗库及降级匹配逻辑见 `js/scenes/poem-scene.js` 中 `POEM_DB` 和 `FALLBACK_POEMS`。

### 成诗呈现

成诗不是普通结果文本。第 3 个字选中后，三枚选中字先形成一条淡淡诗轴，完整短诗再进入诗轴：

1. 3 枚选中字不再停留在水面字阵中，改为诗轴上方的青铜金小字。
2. 完整诗句按两行进入诗轴，优先保证清晰、稳定、不重叠。
3. 诗轴是水面上的半透明绢纸横轴，不做弹窗、卡片或大标题。
4. “再问一诗”时，当前诗句拆成小水点或淡成水痕回声，再让字粒回到可选状态。
5. 同屏始终只保留当前短诗，避免形成诗墙。
6. 成诗态不同时显示水面字阵、入场短句和完成提示，避免任何文字层互相压住。

## 验收要点

- 入场能明确承接第四幕：水痕扩散、字粒入诗，而不是独立选字页。
- 人物只能作为远岸题诗或咏诗的叙事来源，不能过大、不能居中、不能遮挡中央和偏右的水面留白。
- 背景里的竹简、素笺和水面不能出现可读文字，避免和前景字粒、短诗、诗轴冲突。
- 8 个字必须清楚可读，漂浮和水纹不影响辨认。
- 字粒看起来像水中浮字，不像答题按钮、棋子或卡牌。
- 每轮最少 / 最多 3 字成诗；未满 3 字不生成短诗，满 3 字后不追加第 4 字。
- 取消选择和“再问一诗”都温和，不出现错误、失败、扣分或倒计时。
- 同一组选字生成同一条短诗，不接入在线随机生成。
- 每轮短诗进入选中字形成的诗轴，不做普通居中结果文本，也不累积成诗墙。
- 轻知识通过关键词触发，可再次触发，不使用百科卡片。
- 完成态能停留并支持继续组诗：当前短诗、字粒、关键词、“再问一诗”和入口都可读。
- 点击入口后，诗字能自然散成声波，引出远处青铜钟影和第六幕“和鸣”。
- 成诗后短诗以诗轴形式呈现（半透明横轴 + 三字轴首 + 两行排版），不是弹窗、卡片或普通居中文本。
- 本地 `file://` 打开时背景、字粒、短诗和交互都正常显示，不依赖远程资源。
