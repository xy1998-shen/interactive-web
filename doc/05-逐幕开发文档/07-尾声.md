# 07 尾声

## 场景描述

第六幕结束时，三处回声节点的光沿青铜声波向中心回流，"艾、粽、舟、钟"四个小光点短暂出现。第七幕不再重新打开一片大江面，而是承接一张带人物的 ending 背景：帷幔、竹帘和石台纹样退入雾里，左下近景有一位楚地旅人 / 行吟者背向江面而立，手边留着清艾与小舟，中央是一枚由前六幕记忆共同生成的端午印记。

人物不是叙事主角，而是用户走完全程后的视角替身：他站在旅程尽头，看着艾、粽、舟、钟在水雾中合为一印。印记由四层线索组成：左侧艾叶弧线、右侧粽叶折线、下方舟身水痕、外圈钟波圆环。青铜金与雾白微微发光，少量艾草青藏在叶脉和节点里。背景保留第六幕浅色礼乐空间残影，同时下方有极淡楚江水纹反光，表示作品从"礼乐合鸣"落回"楚江记忆"。

这一幕不是分享页，也不是说明页。它是一幅有人的收束长卷：人物给 ending 一个情感落点，中央印记给作品一个记忆落款。用户不再逐个点击节点，而是通过一次连续的"掠过江面"动作让前六幕风物依次归心，最终压缩成四个可以带走的端午符号。

### 入场：三声归印

用户从第六幕点击「入下一幕 ->」后，三处回声节点的光先不硬切消失，而是形成三道淡金声波。声波穿过帷幔和石台纹样，在人物视线前方和中央印记之间汇成四条很淡的光路：

- 艾：从人物身边清艾升起，沿左侧叶弧进入印记。
- 粽：从右侧粽叶折线亮起，沿圆环内侧回到印记。
- 舟：从下方水面与远舟水痕浮起，汇入印记底部。
- 钟：从上方帷幔和青铜纹样间扩散，落成印记外圈钟波。

四条光路不同时运动，避免画面信息过密。入场采用依次归印：艾先从人物身边升起，约 220ms 后粽叶折线亮起，再约 220ms 后舟痕入印，最后钟波落圈。每条光路只走短距离、低透明度、轻微弧线，不做复杂绕行。总时长建议 1.4s 至 1.8s。入场完成后显示动作提示：

> 掠过江面，让四枚印记归心

入场短句：

> 三声归印，楚江入心。

### 连续交互：掠过江面

四个注释点从端午印记中缓慢展开后，不作为独立点击按钮。整张画面接管鼠标 / 指针移动：用户在江面、人物前方或印记周围轻轻掠过，系统按累计移动距离推进"江风入印"进度。

进度分为四段，不显示任务式进度条，只在动作提示中短暂反馈"江风入印 xx%"。每达到 25%、50%、75%、100% 时，系统自动点亮下一枚印记，顺序固定为艾、粽、舟、钟。这样评委不需要寻找小热区，也不会把尾声误解为又一组点击任务。

指针掠过时，画面出现短促金色水痕，随后淡出。水痕只作为动作回声，不常驻铺满画面。

### 第一层：艾

"艾"节点位于圆环左上，靠近人物和清艾所在方向，节点形态像一枚细小叶尖。当掠过进度达到第一段时，人物身边清艾先发出低透明香气线，再沿左侧艾叶弧线回到印记，回忆第二幕悬艾入户。

短句：

> 识一味清艾，愿夏日安康。

轻知识只保留在短句层面，不再扩展药用、辟邪等百科解释，避免尾声变成说明书。

### 第二层：粽

"粽"节点位于圆环右上，贴近背景中粽形和青叶折线所在区域。当掠过进度达到第二段时，右侧粽叶折线沿圆环内侧闭合，少量米粒光点从水面和叶面向内聚拢，回忆第三幕裹青。

短句：

> 裹一叶青愿，把思念包入日常。

此处强调"家常端午"和"思念被包入日常"，不再讲角黍由来，避免和前文重复。

### 第三层：舟

"舟"节点位于圆环下方，和背景水面、小舟、远处龙舟水痕对齐。当掠过进度达到第三段时，水面金线从人物脚边码头、远处舟影和印记底部同时亮起，舟身水痕向中心掠过，带出一圈极淡朱砂鼓波，回忆第四幕听鼓和龙舟竞渡。

短句：

> 听一声江鼓，众桨同心向前。

视觉上只给一次短促动势，随后回到安静，保持尾声"止"的节奏。

### 第四层：钟

"钟"节点位于圆环右下或外圈声波末端，和背景中的钟形纹样、帷幔声波相连。当掠过进度达到第四段时，钟波圆环完整显现，青铜金从外圈向内圈缓慢回流，回忆第六幕礼乐和鸣。

短句：

> 记一段楚风，让礼乐回到江面。

钟不是新增知识点，而是把第六幕的礼乐空间带回楚江水路。

### 完成态：四象成印

四个节点随掠过进度全部点亮后，节点不消失，保持淡金状态。中央端午印记轻微放大再回落，四层线索同时亮起一次；人物衣袂和水面金线只做极轻的回声，不再追加新剧情。随后画面稳定成一幅 ending 长卷：人、江、礼乐空间和端午印记同时成立。

完成短句：

> 沿楚江，寻一缕端午清艾。

完成短句先单独淡入并停留 1 至 2 秒，让用户完成阅读。随后底部中线显示唯一明确操作：

> 重游此程

"重游此程"仍使用全篇一致的江面题签胶囊，不做普通按钮，不加社交分享、二维码、外链或排行榜。用户不点击时，画面保持完成态。

### 重游过渡

用户点击「重游此程」后，端午印记不立刻消失。四个节点先沿圆环逆向淡出，人物与背景不做硬切，只让中央印记变成一圈水纹；水纹向外扩散 800ms 后退回第一幕入江初始态。

这个过渡表达"旅程已完成过的一次回声"，但不写入浏览器存储，不保留历史记录。

## 目的

- 给用户明确完成感，让最后一幕像作品落款，而不是普通信息页。
- 用人物背影建立 ending 的情感视角，让用户感觉旅程有人抵达、有所回望。
- 顺接第六幕"三处回声收束为艾、粽、舟、钟"的过渡，不突然回到无来源的江面。
- 把全篇文化线索压缩成四个可记忆符号：艾、粽、舟、钟。
- 保持尾声信息克制，不新增复杂百科，不扩展二级页面。

## 交互

1. 入场时承接第六幕三处回声，声波回流到人物视线前方，形成四条光路。
2. 四条光路按艾、粽、舟、钟顺序依次归印，每条间隔约 200ms 至 300ms，合成中央端午印记后再出现"艾 / 粽 / 舟 / 钟"四个节点。
3. 用户在画面上掠过江面或移动指针，系统按累计移动距离推进"江风入印"进度。
4. 进度达到 25%、50%、75%、100% 时，按艾、粽、舟、钟自动点亮对应印记；每次点亮只显示对应一行短句。
5. 点亮反馈必须借用背景已有元素：清艾香气、粽叶折线、水面舟痕、帷幔钟波。
6. 已点亮节点保持淡金状态，未点亮节点保持雾白弱提示；节点不作为点击热区。
7. 四个节点全部点亮后，中央印记完成高亮，先出现收束文案。
8. 收束文案淡入后延迟 1 至 2 秒，再出现「重游此程」。
9. 「重游此程」平滑回到第一幕入江，不刷新页面。

## 技术

- 第七幕背景优先使用 `assets/backgrounds/bg-finale-ending-person.png`，以人物、楚乐台余影、水纹和中央印记作为整体构图；如实现成本受限，可降级回 `assets/backgrounds/bg-bell-water.png` 叠加水纹，但必须保留人物或人物视角的 ending 感。
- 背景图中已经包含中央印记雏形；运行时可以在其上叠加 `assets/sprites/sprite-seal.webp` 或 Pixi Graphics 路径，但叠加层要对齐背景印记，不要形成两个中心。
- 节点和路径可用 Pixi Graphics 绘制：艾叶弧线、粽叶折线、舟身水痕、钟波圆环分别作为四层可点亮路径。
- 四个节点按背景内已有视觉方向分布，避免覆盖人物脸部、衣袂和中央印记关键纹样；节点只作为进度反馈，不作为独立点击热区。
- 点亮路径要沿背景已有符号走：清艾到艾节点、粽叶到粽节点、水痕到舟节点、帷幔钟波到钟节点，避免在背景上漂浮一套无来源 UI。
- 注释短句使用 Pixi Text 或 DOM 均可；文字不超过一行，不能使用弹窗和长段说明。短句应放在底部安全区或独立浅底信息带中，不能和四枚印记字牌重叠。
- 掠过推进需要有输入锁和进度阈值，避免快速抖动导致节点状态、短句计时和完成高亮错乱。
- 入场归印和终幕入口出现都需要按时间线控制：光路按 200ms 至 300ms 间隔依次进入印记；「重游此程」晚于收束文案 1 至 2 秒出现。
- 重游操作重置所有场景状态：问诗选字、裹青进度、听鼓节奏、和鸣节点、尾声点亮状态都应清空。
- 页面仍需保持 `file://` 兼容，不引入远程资源、构建步骤或浏览器存储依赖。

## 验收要点

- 入场必须能看出第六幕回声正在收束为尾声印记，不能像硬切到新页面。
- 人物必须融入 ending 氛围，提供情感视角，但不能压过中央端午印记。
- 掠过反馈和点亮反馈必须和背景符号相连，不能像独立 UI 覆盖在背景上。
- 尾声必须有完成感和作品落款感，不像普通信息页或分享页。
- 四个注释点清楚但克制，不能铺成说明书，也不能要求用户逐一寻找点击。
- 四个节点随掠过进度全部点亮前，不出现「重游此程」。
- 重游后所有场景状态能正确复位。
- 页面仍可在本地 `file://` 环境完整运行。
