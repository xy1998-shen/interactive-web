# 逐幕开发文档

本目录用于逐幕推进《楚江寻艾》的设计和开发。每一幕都按同一结构记录：场景描述、目的、交互、技术、素材和验收要点。

开发时优先查看对应分幕文档，再回到以下源文档核对细节：

- `doc/01-项目原则/页面构思.md`
- `doc/01-项目原则/交互串联.md`
- `doc/02-开发规范/技术方案.md`
- `doc/03-设计素材/资产映射与开发准则.md`
- `doc/03-设计素材/设计稿生成规范.md`

## 通用开发原则

- 每幕只保留一个主触发动作，触发后以自动演出完成本幕，避免做成多功能页面。
- 所有素材必须支持本地 `file://` 运行，不依赖远程资源。
- 视觉基调延续明亮雾青江面，不突然进入全黑夜景。
- 轻知识不做百科卡片，优先通过画轴、浮字、小光点或印记注脚出现。
- 完成态必须清楚，并能自然引出下一幕。

## 共用验收项

- 每幕只出现一个主推进入口，入口统一为底部中线的江面题签胶囊。
- 胶囊的位置、尺寸、hover、focus、点击反馈和动效时长在各幕保持一致。
- 胶囊出现时使用 360ms 至 600ms 淡入上浮，消失时使用 300ms 至 400ms 淡出下移，各幕不得自定义另一套节奏。
- 同一时刻画面上可交互的推进入口不超过一个；本幕完成态确立前，推进入口不出现。
- 文案按“动作提示 / 完成短句 / 推进入口”三类使用，不互相混用。
- 画轴只承载叙事和轻知识，不放独立风格的下一幕按钮。
- 首幕推进入口使用 `随舟寻艾 →`，中间幕使用 `入下一幕 →`，终幕使用 `重游此程`。
