# 04 听鼓

## 场景描述

上一幕结尾，窗外出现淡朱砂鼓波。用户点击“入下一幕”后，鼓波沿水纹扩散到楚江水面，画面从临水屋舍回到略深的雾青江面。远山云气和暖金余光仍然可见，江面不进入全黑夜景。

画面采用从船头往前看的鼓手第一视角：下方近景可见龙舟船头、船舷透视线和一面摆在船头的传统鼓，用户像站在船头击鼓；前方江面是龙舟赛水路，两岸有远景围观人群剪影和克制旗影，远处有低对比龙舟竞渡。用户点击鼓面后，鼓波向前方水面扩散，船头、水痕和远处赛道共同产生向前行进的反馈。完成后，龙舟驶过留下长水痕，水痕中浮出字粒，引出下一幕“问诗”。

## 目的

- 形成全篇从静态手作转入江上竞渡的动感高潮。
- 用节奏表现龙舟竞渡和众人合力，但避免做成竞技小游戏。
- 承接第三幕的远处鼓波，不再强行做“粽形变船头”的硬形变。
- 让用户从鼓手第一视角感到“敲一下，江面向前走一步”。
- 用水痕字粒自然引出第五幕“问诗”。

## 交互

1. 入场时，画面显示短句“远鼓入江，舟影渐明。”，总时长控制在 600ms 至 900ms。
2. 用户点击鼓面或水面鼓点区域，触发一次鼓波、短促低频鼓声和第一视角向前推进。
3. 鼓波中短暂浮现“起”“急”“渡”“破”等动作字，随波纹淡出。
4. 第 2、3 次击鼓时，水面边缘短暂浮现两条轻知识：“鼓为令，众桨同频。”、“竞渡从江上追思，成为五月节庆。”。
5. 连续 3 次点击间隔都在 300ms 至 800ms 时，视为稳定节奏，增强桨影、水痕和鼓波半径。
6. 达到内部完成阈值后，远处舟影被鼓声送过江面，画面中央留下透视方向的长水痕。
7. 完成态显示短句“鼓声已渡”，水痕中浮出“江”“艾”“舟”“鼓”“风”等字粒。
8. 水痕旁出现小光点，用户点击后浮现轻知识；不点击也不阻塞进入下一幕。
9. 用户点击「入下一幕 →」后，水痕扩散为深青静水，进入“问诗”。

播放过程中不显示分数、倒计时、排名、失败或“连击”文字。4 次点击作为内部默认阈值，用户看到的是鼓声推进、水路向前和水痕变长，而不是数字进度。

## 技术

- 鼓面点击区域应大于视觉鼓面，保证可用性。
- 鼓波使用 Pixi Graphics 圆环生成，600ms 左右扩散并淡出。
- 船头、远处舟影、背景和水痕使用 GSAP 分段位移 / 缩放，默认累计 4 次点击完成；后续可按体验调整阈值。
- 船头和船舷透视线优先用 Pixi Graphics 代码生成，避免等待新增素材；后续如需更真实木船质感，再补专属船头素材。
- 稳定节奏判定：连续 3 次点击的相邻间隔均在 300ms 至 800ms。
- 稳定节奏只增强视觉表现，不影响完成条件，也不导致失败或倒退。
- 鼓槌使用透明精灵素材，点击时下压撬动鼓面；桨影、水痕、鼓波字和完成字粒优先用 Pixi Graphics / Text 代码生成。
- 鼓面点击时使用 Web Audio 合成约 120ms 的低频包络音；龙舟过江时可叠加一次更开阔的低频衰减。
- 轻知识沿用“随物解锁”：完成后生成水痕光点，点击光点后淡入知识文本。
- 场景完成条件：龙舟过江、水痕和字粒出现、推进入口显示完成。

## 实现经验

本幕包含高频点击、多个 GSAP tween 同时作用于 Pixi 对象，后续维护时优先遵守以下口径：

- 进入场景时记录鼓面和背景的基准变换值，例如鼓面的 `drumBaseScaleX/Y`、`drumBaseY`，背景的 `bgBaseScaleX/Y`、`bgBaseY`。点击反馈、前进视差和完成退场都从基准值计算，不从当前动画中的瞬时值继续叠加。
- 鼓面按压动画每次触发前先 `killTweensOf(drum.scale)`，再用 `fromTo` 从基准缩放压缩回弹，避免快速连击导致鼓面越点越大或越点越扁。
- 鼓槌精灵动画分别补间 `sprite.y / rotation` 和 `sprite.scale.x/y`。Pixi 对象不要使用 DOM 风格的 `scaleX/scaleY` 属性。
- 最后一击会同时触发点击反馈和完成退场。完成退场前先 `killTweensOf(state.drum)` 和 `killTweensOf(state.boatDeckLayer)`，让完成态接管位置和透明度，避免一帧跳动。
- `hitDrum` 中最后一击不再设置“舟影将渡”等过渡 hint，只在 `progress < 1` 时显示“鼓声渐近”，完成文案统一由 `completeDrum` 设置为“鼓声已渡”。
- `spawnOarStrokes` 这类循环创建 Pixi 临时对象的函数，`onComplete` 必须引用当前局部对象并销毁。用 `forEach`、IIFE 或局部函数包住循环体，避免闭包变量错乱。
- `beatWords`、`beatNotes`、`wakeWords` 等配置要允许为空；入口函数先判断数组或文案存在，不让空数组取模、空文案渲染成异常状态。
- `revealDrumKnowledge` 这类完成态函数要做空值检查，尤其是 `knowledgeText` 和 `knowledgeDot`。用户可能从完成态恢复、快速切换场景或走资源降级路径。

## 文案

- 入场短句：远鼓入江，舟影渐明。
- 动作提示：随鼓而行。
- 鼓波动作字：起、急、渡、破。
- 过程轻知识：鼓为令，众桨同频。
- 过程轻知识：竞渡从江上追思，成为五月节庆。
- 核心诗性句：一声鼓起，百桨入江。
- 完成短句：鼓声已渡。
- 轻知识：龙舟竞渡以鼓为令，众桨同频向前。它从江上追思与乡人合力的传说中生长出来，后来成为端午最有节奏感的节庆场面。

### 诗句呈现

第四幕节奏最快，不放长画轴，也不做大段字幕。诗句应跟着鼓波和水痕出现：

1. 每次鼓波只出现 1 个动作字，像鼓声击开水面。
2. 龙舟过江时，“一声鼓起，百桨入江”可以在水痕中短暂闪现。
3. 完成态只保留“鼓声已渡”作为浮字，避免和下一幕问诗抢焦点。
4. 第 2、3 次击鼓时允许出现极短知识浮字，位置避开前景鼓和主入口，不作为常驻说明。
5. 水痕字粒只做第五幕预告，不在本幕组成完整诗句。

## 验收要点

- 入场能明确承接第三幕的淡朱砂鼓波，而不是突兀切到独立竞渡画面。
- 画面仍属于雾青江光体系，不突然全黑，不变成高饱和红色节庆海报。
- 点击鼓面反馈必须即时：鼓面压缩、船头轻推、鼓波、低频鼓声和第一视角推进同步出现。
- 连续 3 次点击间隔都在 300ms 至 800ms 时，应能看到桨影增强和水痕连贯；不满足时仍可正常推进。
- 朱砂红只做节奏点缀，不能把整幕染红。
- 第一视角推进有起鼓、合桨、过江三个层次，但不出现游戏化 UI。
- 轻知识必须通过水痕光点点击解锁，不自动淡入。
- 完成态先显示水痕和字粒，再出现「入下一幕 →」入口，不自动硬切。
- 完成后水痕能明确引出“问诗”的字粒。
- 本地 `file://` 打开时背景、鼓面、龙舟、鼓波、声音降级和字粒都正常可用。
