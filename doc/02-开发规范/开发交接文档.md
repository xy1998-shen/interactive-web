# 《楚江寻艾》开发交接文档

## 1. 项目定位

《楚江寻艾》是一个纯静态交互网页项目，核心体验是一段七幕式端午文化长卷。项目不使用打包工具，不依赖后端接口，所有脚本、样式、字体、图片和音频都以本地相对路径加载。

技术栈：

- PixiJS：全屏 Canvas 渲染、精灵、背景、粒子、波纹和交互区域。
- GSAP：场景动画、过渡、呼吸、淡入淡出和节奏控制。
- 原生 JavaScript：普通 IIFE 模块，通过 `window.ChuJiang` 共享命名空间。
- 原生 DOM/CSS：标题、故事卷轴、提示、进度节点和按钮。

## 2. 运行方式

不要直接用 Chrome 双击或打开根目录 `index.html` 作为验收方式。当前作品使用 PixiJS/WebGL 渲染图片纹理，Chrome 在 `file://` 环境下会阻止本地图片进入 WebGL 纹理，典型报错为 `Failed to execute 'texImage2D' ... image element contains cross-origin data`。

macOS 推荐直接双击根目录的 `启动预览.command`。Windows 推荐直接双击根目录的 `启动预览.bat`。脚本会自动启动本地静态服务并打开浏览器页面；关闭脚本打开的 Terminal 或命令行窗口即可停止服务。

没有 Python 环境时：

- macOS：`启动预览.command` 会按 Python 3、Python、Ruby、PHP 的顺序寻找可用运行环境；通常系统自带 Ruby 或 PHP 时也能直接启动。
- Windows：`启动预览.bat` 会优先使用 Python 3；如果没有 Python，会自动调用 `tools/windows-static-server.ps1` 提供的 PowerShell 静态服务。若公司策略禁用了 PowerShell 脚本，再安装 Python 3 后重试。

也可以手动启动静态服务：

```bash
python3 -m http.server 39217
```

访问：

```text
http://127.0.0.1:39217/
```

验收时优先确认：

- 页面标题显示为《楚江寻艾》。
- 首屏能看到 Canvas 背景、DOM 标题和江流进度。
- 浏览器控制台没有资源 404、脚本报错或字体路径错误。
- 断网后通过本地服务仍能打开，不能依赖外部 CDN、接口或远程素材。


根目录只保留运行入口、源码、运行资源、依赖库和必要文档。过程目录如 `tmp/`、`assets/source/`、`.omc/`、编辑器状态目录不应重新提交。

## 4. 入口与加载顺序

`index.html` 是唯一入口，当前脚本加载顺序不能随意调整：

1. `libs/pixi.min.js`
2. `libs/gsap.min.js`
3. `js/constants.js`
4. `js/config.js`
5. `js/utils.js`
6. `js/audio-manager.js`
7. `js/asset-loader.js`
8. `js/dom-layer.js`
9. `js/scenes/empty-scene.js`
10. `js/scenes/base-scene.js`
11. 各分幕场景脚本
12. `js/scene-manager.js`
13. `js/app.js`

所有业务脚本都挂在 `window.ChuJiang` 下。因为没有模块系统，后加载文件可以读取先加载文件暴露的 `NS.*`，但反过来不行。

## 5. CSS 文件

`css/style.css` 负责全部 DOM 层视觉和字体声明：

- `@font-face`：注册本地中文字体。
- `#stage`：全屏 Pixi Canvas。
- `.dom-layer`：覆盖在 Canvas 上方的 DOM 交互/展示层。
- `.hero-copy`、`.chapter-mark`、`.hint`：首屏与章节提示。
- `.story-scroll`、`.intro-cta`：第一幕故事卷轴与进入下一幕按钮。
- `.river-progress`、`.progress-node`：底部江流进度节点。

Canvas 内部的背景、精灵、粒子和大部分场景特效不在 CSS 里实现。

## 6. JavaScript 文件职责

### `js/constants.js`

全局常量入口：

- `NS.SCENES`：七幕顺序、章节编号、展示名称、每幕预加载资源 key。
- `NS.FONT_STACKS`：Pixi Text 使用的字体栈。
- `NS.ASSET_MANIFEST`：运行素材 key 到真实路径的映射。
- `NS.TITLE_ASSETS`：首屏和章节标题艺术字路径。

新增运行图片、纹理、音频以外的标题图时，优先从这里登记。

### `js/config.js`

全局数值配置入口，按模块和场景分块：

- `renderer`：Pixi 分辨率和帧时间保护。
- `audio`：背景音乐路径、基础音量、各场景音量。
- `intro`、`mugwort`、`wrap`、`drum`、`poem`、`bell`、`finale`：各幕布局、阈值、动画时长、文字和颜色。

后续调交互手感、位置、节奏和文案时，优先改这里，减少场景代码硬编码。

### `js/app.js`

应用启动器：

- 创建 Pixi Application。
- 初始化 DOMLayer、AssetLoader、AudioManager。
- 绑定底部进度节点和第一幕 CTA。
- 绑定 Pixi ticker，逐帧调用当前场景 `onUpdate(dt)`。
- 加载第一幕资源并进入初始场景。

### `js/scene-manager.js`

场景生命周期管理：

- `goTo(index)`：加载目标场景资源，进入目标场景。
- `enterScene(index, sceneMeta)`：退出旧场景，重置 DOM，创建并进入新场景。
- `preloadNext(index)`：预加载下一幕资源。
- `completeCurrent()`：标记当前场景完成，并更新进度节点。
- `update(dt)`：把帧更新转发给当前场景。

### `js/asset-loader.js`

资源加载器：

- 根据 `NS.ASSET_MANIFEST` 按 key 加载图片。
- 缓存 `Image` 和 `PIXI.Texture`。
- 资源失败时生成 fallback canvas，避免页面完全中断。

### `js/dom-layer.js`

DOM 浮层控制器：

- 管理章节编号、章节名、主标题、副标题和提示文案。
- 管理故事卷轴、第一幕 CTA、进度节点状态。
- 加载标题艺术字，并在资源缺失时回退到文字。
- 向场景暴露 action handler，支持按钮触发当前场景逻辑。

### `js/audio-manager.js`

背景音乐控制：

- 用户首次 pointer / keydown 后解锁播放。
- 按场景切换目标音量。
- 提供 `duck()`，在鼓声、钟声等局部音效出现时临时压低背景音乐。

### `js/utils.js`

通用工具函数集合，供场景和基础模块复用。维护时优先放无业务状态、无 DOM 副作用的小工具。

## 7. 场景文件职责

所有场景都在 `js/scenes/` 下。

- `base-scene.js`：MVPScene 基类和公共能力，包含背景铺放、知识点、面板式场景、公共完成态等。
- `empty-scene.js`：缺少实现时的兜底场景。
- `intro-scene.js`：第一幕入江，负责拨雾、故事卷轴、随舟寻艾入口。
- `mugwort-scene.js`：第二幕寻艾，负责艾草入户和四个艾俗面板。
- `wrap-scene.js`：第三幕裹青，负责粽叶、束线、粽俗面板和完成态。
- `drum-scene.js`：第四幕听鼓，负责第一视角鼓序列、点击反馈和场景推进。
- `drum-effects.js`：听鼓相关水波、鼓波、粒子等效果补充。
- `poem-scene.js`：第五幕问诗，负责字粒选择、诗句生成和诗意反馈。
- `bell-scene.js`：第六幕和鸣，负责三处回声、声路、编钟反馈和完成态。
- `finale-scene.js`：第七幕尾声，负责连续掠过江面、四象归印、总结面板和重游。

当前 `scene-manager.js` 在非第一幕时创建 `NS.MVPScene`，各分幕脚本通过扩展基类或挂载方法实现对应场景能力。修改某一幕前，先阅读对应 `doc/03-逐幕剧本/*.md`。

## 8. 资源目录

### `assets/backgrounds/`

场景背景和面板背景。主要由 `NS.ASSET_MANIFEST` 引用，如入江江面、寻艾村庄、裹青桌面、听鼓第一视角、问诗背景、和鸣背景、尾声背景和各幕面板图。

### `assets/sprites/`

透明精灵和交互元素，如艾草、粽叶、粽子、龙舟、编钟、印记、诗签和气味/蒸汽/墨波等。

### `assets/textures/`

低频纹理，目前保留水纹和端午暗纹。

### `assets/titles/`

透明标题艺术字。DOM 里必须保留等价文本，标题图只作为视觉增强，不能作为唯一信息来源。

### `assets/fonts/`

本地字体文件和授权说明。CSS 通过相对路径加载：

- `NotoSerifCJKsc-subset.woff2`
- `LXGWNeoZhiSong-subset.woff2`
- `LXGWWenKai-subset.woff2`
- `LXGWWenKaiScreen-subset.woff2`
- `OFL-LXGWWenKaiTC.txt`

### `assets/audio/`

背景音乐，目前为 `mist-river-recall.mp3`。

### `assets/prompt-log.md`

素材生成和替换记录。当前项目不保留源稿目录，prompt-log 只作为追溯说明，不应再让运行逻辑依赖其中的历史源文件路径。

## 10. 常见开发任务

### 新增或替换素材

1. 放入 `assets/backgrounds/`、`assets/sprites/`、`assets/textures/` 或 `assets/titles/`。
2. 在 `js/constants.js` 登记资源 key。
3. 在对应 `NS.SCENES[*].assets` 中加入预加载 key。
4. 在场景代码中通过 `this.app.assets.load(key)` 或已有封装使用。
5. 更新对应逐幕文档和 `README.md` 的资源说明。

### 调整某一幕交互

1. 先看 `doc/03-逐幕剧本/对应幕.md`。
2. 优先修改 `js/config.js` 中对应场景配置。
3. 必要时修改 `js/scenes/对应场景.js`。
4. 验证场景进入、完成态、进度节点、下一幕预加载。

### 新增场景

1. 在 `NS.SCENES` 中新增场景元数据。
2. 在 `NS.ASSET_MANIFEST` 中登记资源。
3. 新增 `js/scenes/*.js`，并在 `index.html` 中按依赖顺序加载。
4. 更新 `scene-manager.js` 的场景创建逻辑。
5. 补充逐幕文档。

## 11. 验收清单

- `index.html` 可直接打开。
- `python3 -m http.server 39217` 下访问正常。
- 控制台没有 error。
- `js/constants.js`、`js/config.js`、`css/style.css`、`index.html` 引用的文件全部存在。
- 底部 7 个进度节点能切换或随场景完成更新。
- 背景音乐在首次用户交互后播放，不交互时不阻塞视觉体验。
- 不提交 `tmp/`、`assets/source/`、`.omc/`、`.DS_Store`、编辑器状态目录。

## 12. 当前维护重点

- 包体积主要集中在字体和 PNG 背景，后续若有提交大小限制，应优先做字体子集和背景 WebP 压缩。
- 场景脚本通过全局命名空间协作，新增文件时要特别注意 `index.html` 的加载顺序。
- 第七幕尾声采用连续掠过江面的交互，不应退回点击节点式推进。
- 第四幕听鼓当前只使用 `drum-bg-firstperson.png` 作为主背景，不再恢复旧拆层鼓面/鼓槌方案。
