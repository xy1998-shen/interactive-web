# 《楚江寻艾》技术方案

## 一、技术选型总览

| 层级 | 选择 | 说明 |
|------|------|------|
| 渲染引擎 | PixiJS v7 | WebGL 2D 渲染，内置精灵交互、粒子容器、滤镜（模糊/置换贴图）、遮罩。通过 `<script>` 标签引入本地 UMD 文件 |
| 动画库 | GSAP 3 | 时间线编排、缓动、属性补间。处理场景过渡和元素变形。通过 `<script>` 标签引入 |
| 音频 | Web Audio API（原生） | 不依赖第三方库。用振荡器和噪声合成水声、鼓点、编钟，零音频文件依赖 |
| 构建工具 | 无强制构建步骤 | 当前提交形态为静态站，直接通过普通 `<script>` 顺序加载本地 UMD 库和 IIFE 业务模块 |
| 语言 | JavaScript | 业务代码用 IIFE 自执行函数封装，不依赖 ES Modules（兼容 file:// 协议） |
| 字体与标题 | `ChuJiangWenKai` / `ChuJiangCalligraphy` / 透明标题 WebP | 正文和 DOM 长文本用文楷子集与系统宋体 fallback；章节题签、问诗诗轴用书法子集；主标题、章节大字和少量动作短句用透明标题图，但必须保留文本 fallback |

## 二、为什么选这套方案

### 选 PixiJS 而非其他

- **不选 Three.js**：本作品全程 2D，Three.js 增加 600KB+ 体积和 3D 复杂度，零收益。
- **不选纯 Canvas 2D**：手写拖拽命中检测、精灵批处理、滤镜实现至少多花 3 天。PixiJS 开箱即用。
- **不选 SVG + CSS**：雾气粒子、水纹扩散、连续轨迹等效果在 DOM 中性能差，100+ 动画元素时掉帧严重。
- **不选 React/Vue**：单页艺术作品无路由、无状态绑定需求，框架只增加体积和复杂度。

### 选 GSAP 而非 CSS 动画

- 需要精确的时间线编排（场景进入 800ms→交互反馈 150ms→文字浮现 600ms）。
- 需要元素变形（路径插值）。
- CSS 动画无法做到复杂序列编排和动态参数控制。

### 选 Web Audio API 而非音频文件

- 设计文档要求"不依赖预录音频"，轻量模拟优先。
- 合成方案零文件依赖，不增加包体积。
- 鼓点、编钟、水声均可用简单振荡器+滤镜实现。
- 完全可控：音量、音高、时长均可随交互动态调整。

## 三、file:// 兼容性优先策略

**最高优先级约束**：提交物必须在 `file://` 协议下双击 index.html 即可运行，零服务器依赖。

这意味着：
- **禁止 `<script type="module">`**：ES Modules 在 file:// 下触发 CORS 错误。
- **禁止 `import`/`export` 语法出现在最终产物中**。
- **禁止 fetch/XHR 加载本地文件**：file:// 下跨文件请求被阻止。
- **所有资源通过 `<script>`、`<link>`、`<img>` 等标签以相对路径引入**。

### 开发与提交模式

当前项目不要求构建步骤，源码即提交运行形态。入口通过 `index.html` 按顺序加载 `libs/`、`js/constants.js`、`js/config.js`、工具模块、场景模块、`scene-manager.js` 和 `app.js`。

后续如果为了开发效率引入 Vite 或其他构建工具，最终产物仍必须是非 module 的 IIFE / UMD 形态，并保留本地相对路径、无网络依赖和 `file://` 双击运行能力。

### file:// 验收作为每日首项检查

每天开发结束后，必须执行：
1. 检查入口、CSS、JS 和资源清单中的相对路径真实存在。
2. 断网。
3. 在 Finder 中双击 `index.html` 用浏览器打开。
4. 确认完整体验无报错；如使用本地静态服务辅助调试，也不能替代 `file://` 验收。

**此验收优先级高于任何功能开发。**

## 四、渲染架构

采用 **PixiJS 画布 + DOM 浮层** 混合方案：

```
┌──────────────────────────────────┐
│  DOM 浮层（文字、题签、提示）      │  pointer-events: none（选择性穿透）
├──────────────────────────────────┤
│  PixiJS 画布（全屏）              │  处理所有视觉效果和交互
│  - 背景层：水面、雾气、远景       │
│  - 交互层：精灵、粒子、轨迹       │
│  - 特效层：波纹、声波、过渡       │
└──────────────────────────────────┘
```

- **PixiJS 画布**：处理雾气、水纹、粒子、精灵拖拽、波纹扩散等全部视觉效果。单一 WebGL 上下文保证性能。
- **DOM 浮层**：标题、短句、题签、知识点用 HTML 渲染。中文字体在 DOM 中显示效果远优于纹理渲染。
- **交互事件**：全部由 PixiJS 的 `eventMode` 捕获，DOM 浮层设为 `pointer-events: none`（需要点击的按钮除外）。

### 字体与标题资源规则

- `ChuJiangSerif` 来源于 `assets/fonts/NotoSerifCJKsc-subset.woff2`，用于正文、画轴、知识点等长文本。CSS fallback 保持 `"Songti SC", "STSong", "FangSong", "Noto Serif CJK SC", serif`。
- `ChuJiangZhiSong` 来源于 `assets/fonts/LXGWNeoZhiSong-subset.woff2`，用于章节题签和问诗成诗态；`ChuJiangWenKai` / `ChuJiangWenKaiScreen` 来源于 `assets/fonts/LXGWWenKai-subset.woff2`、`assets/fonts/LXGWWenKaiScreen-subset.woff2`。
- 问诗短诗、诗轴轴首和诗意浮字不要使用 `ChuJiangWenKai`，避免儿童感；fallback 使用 `STKaiti, Kaiti SC, KaiTi, Songti SC, STSong, Noto Serif CJK SC, serif`。
- `assets/titles/*.webp` 只作为视觉层：首屏主标题、7 个章节大字、`拨开江雾`、`入江寻艾`、`随舟寻艾`。DOM 文本、`aria-label` 或 Pixi Text fallback 必须保留，资源缺失时页面仍可读。
- 标题图必须人工校字，错字、异体误用、严重粘连一律废弃；正文、按钮、知识点、尾声注释不做成图片。

## 五、核心模块设计

### 1. 场景管理器（SceneManager）

负责场景生命周期和切换：

```
场景生命周期：
  onEnter(fromScene)  → 构建显示对象、进入动画、注册交互
  onUpdate(dt)        → 每帧逻辑（粒子运动、环境动画）
  onComplete()        → 达成目标、触发过渡
  onExit(toScene)     → 退出动画、清理监听器

切换流程：
  当前场景.onExit() → 过渡动画 → 下一场景.onEnter()
```

预加载策略：用户体验场景 N 时，后台静默加载场景 N+1 的素材。

生命周期规则：

- 场景内的 GSAP 时间线、无限循环 tween、事件监听和延迟回调都必须进入 `cleanups`。
- 延迟回调优先使用 `MVPScene.scheduleCall()`，由公共基类在 `onExit()` 时统一 kill，并在回调执行前检查场景容器仍存在。
- 场景切换前必须释放当前场景持有的音频上下文、临时 Pixi 对象和 DOM 行为入口，避免旧场景继续改写完成态或跳转。

### 2. 音频引擎（AudioEngine）

四种合成音色：

| 音色 | 合成方式 | 触发时机 |
|------|----------|----------|
| 江水环境声 | 棕色噪声 + 带通滤波(200-600Hz) + 增益LFO(0.1Hz) | 全程低音量循环 |
| 鼓点 | 正弦振荡器 80Hz + 指数衰减 120ms | 听鼓场景点击时 |
| 编钟 | 三谐波叠加(440/880/1320Hz) + 2s 指数衰减 | 和鸣场景轻触时 |
| 艾草沙沙 | 白噪声 + 高通滤波 2000Hz + 50ms 门控 | 寻艾场景拖动时 |

音频策略：
- 默认静音，首次交互后显示"启声"按钮。
- AudioContext 在第一次指针事件中创建（符合浏览器自动播放策略）。
- 声音可关闭，关闭后纯视觉体验完整不受影响。
- 场景自建的 AudioContext 必须在 `onExit()` 中关闭；公共音频能力应由基类统一管理，避免多次进入场景后累积上下文。

### 3. 粒子池（ParticlePool）

预分配对象池，避免运行时 GC：

- 预创建固定数量精灵（如 80 个轨迹粒子）。
- 激活时设置位置和透明度，用 GSAP 补间淡出。
- 淡出完成后回收到池中复用。
- 每个场景设独立粒子预算（见性能预算章节）。

### 4. 过渡引擎（TransitionEngine）

三级过渡策略（按优先级递减）：

- **一级：路径变形**：定义起止形状为等点数 SVG 路径，GSAP 补间路径数值，每帧重绘。用于叶脉→粽叶、粽形→龙舟等几何过渡。
- **二级：增强渐隐 + 粒子轨迹**：上一幕关键元素沿预设轨迹飞向下一幕目标位置，配合透明度渐变和 SVG 线条重绘。视觉效果好，体积零增加。
- **三级（最后手段）：序列帧**：仅在前两级均不可行时使用。单组不超过 12 帧，每帧 < 30KB，严格控制总增量 < 300KB。

## 六、关键场景技术实现

### 入江：雾气消散

```
实现思路：
1. 创建 RenderTexture 作为雾气遮罩，初始填充白色（全雾）
2. 用户拖动时，在遮罩上绘制黑色圆（半径随机变化，模拟自然感）
3. 将此纹理作为雾气图层的遮罩 → 黑色区域透明（雾散开）
4. 雾气图层叠加置换滤镜（滚动噪声纹理驱动）→ 雾有机流动
5. 追踪已清除面积比例 → 超过 60% 触发场景完成
```

### 寻艾：拖艾入户

```
实现思路：
1. 艾草精灵设置 eventMode: 'static'，监听 pointerdown/move/up
2. 拖动时每帧从粒子池激活一个圆形粒子，放置在指针位置
3. 粒子颜色 #8EA86A，透明度从 0.7 补间到 0，持续 800ms
4. 定义窗前和门楣目标区域（圆形或矩形命中检测），距离 < 阈值时吸附锁定
5. 锁定后播放人物抬手挂艾动画、门窗光效和轻知识文字
```

### 听鼓：节奏推进

```
实现思路：
1. 鼓面精灵覆盖画面下方 40% 区域，点击即触发
2. 每次点击：生成朱砂红圆环向外扩散（600ms 淡出）+ 鼓声合成
3. 龙舟精灵从左向右，每次点击 GSAP.to 向右移动一段（ease: power2.out）
4. 连击加成：400ms 内连续点击，波纹半径 ×1.2，龙舟步进加大
5. 累计 8-12 次点击，龙舟驶出右边界 → 场景完成
```

### 裹青：折叠交互

```
实现思路（简化版，确保可落地）：
1. 两片粽叶精灵分居左右
2. 步骤一：拖动左叶尖端到中心吸附点 → 叶片旋转 35° 对齐
3. 步骤二：拖动右叶尖端对称吸附 → 两叶重叠
4. 步骤三：线条精灵出现，横向拖动触发自动缠绕动画
5. 米粒粒子在折叠过程中被吸向中心（引力补间）
6. 无失败状态：未完成的拖动弹回，目标点脉冲提示

降级方案：如果几何交互手感不好，改为"拖动叶片到中心，自动播放折叠动画"。
视觉效果一致，但省去约束几何的复杂度。
```

### 问诗：触字成句

```
实现思路：
1. 8 个文字精灵浮在水面（正弦波缓慢上下浮动）
2. 可选文字：江、艾、舟、鼓、风、月、兰、钟
3. 悬停时光晕变亮，点击后上浮并发光
4. 选满 3 个字后，从预设诗句表中确定性匹配
5. 匹配逻辑（严格确定性，禁止随机）：
   a. 将选中 3 字按固定字典序排列生成 key（如"艾江舟"）
   b. 精确 key 命中预设表 → 使用对应诗句
   c. 未命中 → 遍历预设表，找到"包含选中 3 字中任意 2 字"的首条 → 使用
   d. 仍未命中 → 使用第一个选中字对应的固定兜底句（每字1首，共8首）
6. 匹配到的诗句以纵向排列浮现在画面中

预设 20 首诗覆盖高频组合 + 8 首兜底句（每个字各一首），确保任何选择都有确定输出。
同一组合永远返回同一首诗，用户可预期结果。
```

### 和鸣：声波扩散

```
实现思路：
1. 编钟剪影精灵居画面上方
2. 点击触发：3-5 个同心金色圆环依次扩散（每环延迟 200ms）
3. 同步触发编钟合成音（三谐波 + 2s 衰减）
4. 扩散过程中，前几幕元素以 15% 透明度短暂闪回
5. 点击 3 次后，声波自动向内收缩 → 汇聚成圆形 → 过渡到尾声
```

## 七、性能预算

### 包体积目标

所有大小均为**解压后实际文件大小**（非 gzip）：

| 组成 | 目标大小（解压后） | 说明 |
|------|----------|------|
| libs/（PixiJS + GSAP） | ~ 560KB | UMD 压缩版，固定不变 |
| js/（业务代码） | < 300KB | 多个 IIFE 文件，按职责拆分 |
| CSS | < 30KB | |
| 字体 | 按实际授权和字形需求控制 | 当前使用完整 OTF / TTF 和系统 fallback；后续可重新生成 WOFF2 子集 |
| 运行图片素材 | 按比赛限制压缩 | 主要体积来源 |
| 提交包 | 以比赛要求为准 | 只包含运行必需资源，不包含过程材料 |

当前工作目录已清理 `已清理源稿目录/`、`tmp/`、`.omc/` 等过程材料；最终提交包仍应只包含入口、样式、脚本、运行资源、依赖库和必要说明文档。

### 运行时目标

| 指标 | 目标 | 实现方式 |
|------|------|----------|
| 交互帧率 | 60fps | 对象池、粒子数量上限、GPU 合成 |
| 过渡帧率 | 45fps 以上 | 过渡期间减少活跃粒子 |
| 首次有意义渲染 | < 500ms（本地文件） | 启动只加载场景 1 素材 |
| 输入到视觉反馈 | < 16ms（1帧） | 指针事件直接处理，不防抖 |
| 场景切换加载 | < 100ms | 预加载下一场景素材 |

### 各场景粒子预算

| 场景 | 最大粒子数 | 说明 |
|------|-----------|------|
| 入江 | 200 雾粒 + 50 水纹 | 雾粒用大精灵少数量 |
| 寻艾 | 80 轨迹粒子 | 短生命周期 |
| 裹青 | 40 米粒 + 20 文字 | 极少 |
| 听鼓 | 30 鼓波 + 60 水花 | 波为简单扩散圆 |
| 问诗 | 8 文字 + 30 环境粒子 | 最轻量 |
| 和鸣 | 15 声波环 + 50 叠合 | 环靠缩放而非数量 |
| 尾声 | 30 汇聚元素 | 一次性动画 |

## 九、开发里程碑（14 天）

## 十、风险预案

| 风险 | 应对方案 |
|------|----------|
| 裹青折叠手感差 | 改为单步拖拽+自动折叠动画 |
| 变形过渡实现困难 | 降级为"增强渐隐 + 粒子轨迹 + SVG 线条重绘"，不使用序列帧 |
| AI 生成素材风格不一致 | 严格 prompt 模板 + 统一色彩后处理 |
| 包体积超标 | 压缩图片质量、减少背景图层、合并纹理 |
| WebGL 不可用 | 目标为 PC 端现代浏览器，WebGL 覆盖率 > 99%。极端情况下显示静态提示页引导用户更换浏览器，不承诺 Canvas 2D 自动降级 |
| file:// 协议加载异常 | 每日构建后断网验证，禁止 ES Modules / fetch 本地文件 |
| 时间不够 | 保证 7 幕主流程可进入、可完成、可收束；高级变形、移动端适配和声音精修可降级 |
| 浏览器禁止音频 | 默认静音设计，视觉体验独立完整 |
