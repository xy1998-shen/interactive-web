# 《楚江寻艾》设计稿生成规范

## 一、设计稿目标

本阶段使用 GPT 生成 **PC 端 16:9 中文设计方向稿**，用于确定整体画面、构图、色彩、氛围、交互对象位置和中文标题气质。

设计稿不是最终切图，不要求一次性生成可直接开发的 UI。它的作用是：

1. 统一作品视觉气质，避免后续素材和页面各做各的。
2. 明确每一幕的主视觉、留白、交互对象和中文文字位置。
3. 为后续素材拆分提供依据。
4. 让开发前先看到完整体验的视觉节奏。

当前只考虑 **PC 端横屏 16:9**，移动端适配暂缓。

## 二、中文文字生成策略

设计稿需要出现中文，但要控制文字量。

允许生成：

- 主标题：`楚江寻艾`
- 副标题：`一场屈原故里端午的沉浸式交互诗`
- 章节名：`入江`、`寻艾`、`裹青`、`听鼓`、`问诗`、`和鸣`、`端午印记`
- 每幕短句：
  - `拨开江雾`
  - `艾气入户，清风守门`
  - `青叶合拢，江心有愿`
  - `一声鼓起，百桨入江`
  - `字从水起，风过楚江`
  - `金声入水，楚江和鸣`
  - `沿楚江，寻一缕端午清艾`
- 尾声印记注释：
  - `识一味清艾`
  - `裹一叶青愿`
  - `听一声江鼓`
  - `记一段楚风`

不建议生成：

- 大段说明文字。
- 知识点正文。
- 导航菜单文字。
- 按钮文字。
- 英文装饰字。

注意：GPT 生成中文仍可能有错字。设计稿中的中文用于视觉预览，最终网页正文、知识点、按钮和题签仍以 DOM 实现为准。筛选设计稿时，凡是标题或短句错字明显的版本直接淘汰。

### 设计稿字体与标题素材策略

设计稿里的书写感标题可以转化为少量透明 PNG / WebP 标题素材，用来保留 AI 生成图中的笔触气质，但只适用于以下位置：

- 首屏主标题 `楚江寻艾`。
- 每幕大号章节字：`入江`、`寻艾`、`裹青`、`听鼓`、`问诗`、`和鸣`、`端午印记`。
- 极短装饰性短句，且必须经过人工校字。

不适合切成图片的文字：

- 副标题、知识点、画轴正文、按钮文案、进度节点、无障碍标签。
- 需要响应式换行或后续频繁修改的文案。

标题图片素材要求：

- 文件放入 `assets/titles/`，命名为 `title-scene-id.webp` 或 `title-scene-id.png`。
- 优先透明背景，保留文字边缘柔光，不带整张设计稿背景。
- 同时保留 DOM 文本作为可访问 fallback，图片只承担视觉笔触。
- 每个标题素材必须人工检查错字、缺笔、粘连和不可读问题。

## 三、生成顺序

### 第一轮：3 张风格定调稿

先生成三张，确定整体审美：

1. **封面 / 入江**：第一眼沉浸感。
2. **听鼓 / 龙舟高潮**：动态和节奏感。
3. **和鸣 / 尾声印记**：最终收束和高级感。

这三张如果风格不统一，先调整提示词，不继续生成 7 幕。

### 第二轮：7 张分幕静态稿

生成完整页面分幕：

1. 入江
2. 寻艾
3. 裹青
4. 听鼓
5. 问诗
6. 和鸣
7. 尾声

每张只需要表现静态布局：背景、主体、交互对象、中文标题/短句位置、章节题签位置。

### 第三轮：交互动效分镜

每一幕生成 3 帧：

1. 初始态
2. 交互中
3. 完成态

用于指导动效，不作为最终素材直接使用。

## 四、通用生成约束

所有设计稿统一使用以下约束：

- 画幅：16:9 横屏，PC 网页首屏设计。
- 风格：诗意东方、雾青江面、水墨质感、电影感留白、克制高级；以 `design/01.png` 的明亮雾青江光为视觉锚点。
- 色彩：晨雾青、浅水青、云水米白、雾白为主，楚江青黑和深墨绿用于深水与阴影，艾草青、朱砂红、青铜金、夕照暖金为少量点缀。
- 界面：不要顶部导航栏，不要普通网页按钮，不要卡片式布局。
- 中文：只生成指定中文标题、章节名或短句，不生成额外文字。
- 交互提示：用光点、水纹、题签、轨迹暗示，不使用现代 UI 图标；用户轻点后进入自动演出，不要求连续拖动。
- 文字说明交互：用物件旁小光点、关键词水纹、印记节点表现，不要画成弹窗、侧栏或百科卡片。
- 禁止：外链、二维码、logo、水印、签名、真实浏览器框、手机界面、现代建筑、卡通节日风。

统一负面约束：

```text
不要生成多余中文、不要生成英文、不要生成段落说明、不要生成网页导航栏、
不要生成浏览器外框、不要生成普通按钮、不要生成卡片布局、不要生成二维码、
不要生成 logo、水印、签名、现代 UI 图标、图库照片风、卡通风、可爱游戏风、
赛博霓虹、明亮节庆海报色、恐怖元素、鬼怪意象、骷髅、血迹、暴力画面。
```

## 六、第一轮：风格定调稿 Prompt

### 1. 封面 / 入江

用途：确定第一眼沉浸感、标题区域、江雾和入口动作。

```text
请生成一张 PC 端 16:9 横屏网页封面设计稿，作品名称为《楚江寻艾》。

场景：封面 / 入江。整屏是一片清晨或暮色中的楚江水面，江面是雾青色而不是全黑。薄雾在江面低低流动，但不要遮满画面。远处偏侧有一叶很小的舟或龙舟剪影，若隐若现，制造“江上将行”的悬念。画面中下部有一圈很淡的水纹和微弱光点，暗示用户可以拖动水面、拨开江雾进入旅程。雾中可以隐约出现艾草叶影，但不要明显。

视觉风格：诗意东方、水墨肌理、雾青江面、柔和薄雾、克制的电影感光影、留白充足、高级安静、沉浸式网页首屏。整体比深夜画面更明亮，有清透江光，但不能变成旅游风景照。

色彩：楚江青黑 #101C1B 和深墨绿 #1D332C 做水面深处与阴影，雾白 #E8E1D2、晨雾青 #D8DED1、浅水青 #9EAEA3 做江面亮部和薄雾，少量艾草青 #8EA86A 做隐藏叶影。

构图：全屏开阔江面背景，标题位于画面中央偏上，远舟位于画面深处偏左或偏右，尺寸很小，只制造悬念；水纹交互区域位于画面中下部；不要卡片，不要菜单，不要普通按钮。
```

## 九、设计稿评审标准

每张设计稿生成后按以下标准判断，低于 20 分不进入下一步：

| 项目 | 评分标准 |
| --- | --- |
| 主题准确 | 是否能看出端午、楚江、荆楚文化相关元素 |
| 风格统一 | 是否符合 ../01-项目原则/设计语言.md，不像旅游官网或国潮海报 |
| 中文准确 | 指定中文是否清晰、无错字、无多余文字 |
| 构图可开发 | 主视觉、交互对象、文字区域是否清楚 |
| 沉浸克制 | 是否全屏沉浸、少按钮、少装饰、画面不拥挤 |

淘汰标准：

- 出现错字、乱码、伪中文、额外英文。
- 出现普通导航栏、按钮、卡片布局。
- 风格太卡通、太亮、太商业海报。
- 主体太写实，像旅游照片或博物馆照片。
- 交互对象不清楚，开发无法拆解。
- 每一幕之间风格差异过大。

## 十、设计稿到素材的拆解规则

设计稿通过后，不直接把整张图当页面背景使用，应拆解为：

- 背景层：雾青江面、远山云气、江岸浅影、深青静水、竹简暗影、雾气。
- 交互精灵：艾草、粽叶、鼓面、龙舟、编钟、印记。
- 特效纹理：水纹、雾气、声波、暗纹。
- DOM 文字：标题、短句、知识点、题签最终仍由网页实现。

拆解时遵守：

- 背景可以从设计稿衍生，但必须保留交互留白。
- 精灵必须重新生成透明背景版本，不从完整设计稿里硬裁。
- 设计稿里的中文只作为排版参考，不作为最终文字图片切出。
- 每个素材继续按 `素材管理.md` 和 `提示词管理.md` 入库。
