# 《楚江寻艾》资产映射与开发准则

## 一、文档目的

本文用于把已生成的设计稿和素材固定成开发执行清单，避免后续实现时出现以下问题：

- 把设计稿整图直接当页面背景，导致交互对象无法拆分。
- 每一幕临时换图，破坏统一风格。
- 素材用途不清，重复加载或错用背景。
- 文字说明变成普通卡片、弹窗或百科页面。
- 开发阶段为了省事偏离 `../01-项目原则/设计语言.md`、`../01-项目原则/页面构思.md`、`../01-项目原则/交互串联.md`。

开发时优先遵循本文的资产映射。如果后续需要替换素材，必须先更新本文，再进入实现。

## 三、全局资产使用原则

1. **设计稿用于参考，素材用于实现**。
   第二轮和第三轮 PNG 不进入最终页面资源加载清单。

2. **背景按层叠使用**。
   每幕至少包含：主背景、纹理 / 雾气叠层、交互精灵、DOM 文字层。

3. **正文文字全部用 DOM 实现，标题可少量图片化**。
   副标题、短句、知识点、题签、问诗字粒、尾声注释都不从设计稿切图。首屏主标题和各幕大号章节字允许使用透明 PNG / WebP 标题素材，但必须保留 DOM fallback。

4. **精灵不做永久固定图标**。
   精灵必须参与交互、动效或过渡，例如点击、飞行、发光、回声、收束。

5. **纹理只做气氛，不做主视觉**。
   `tex-water-ripple.webp`、`tex-pattern-duanwu.webp` 透明度要低，避免抢交互对象。

6. **深色素材谨慎使用**。
   后段可以用 `bg-river-dark.webp` 或深色遮罩，但全篇仍要保持“雾青江光”基调。

## 五、全局 UI / DOM 映射

| UI 元素 | 实现方式 | 风格要求 |
| --- | --- | --- |
| 主标题 | 透明标题素材 + DOM fallback | 居中偏上，书写感，人工校字 |
| 章节题签 | DOM 文本 + 小线条 | 低透明，不做导航栏 |
| 动作提示 | DOM 文本 | 2-6 字，靠近交互区域 |
| 轻知识 | DOM 文本 | 通过小光点 / 关键词 / 印记节点触发 |
| 问诗字粒 | DOM 或 Pixi Text | 可点击、漂浮、发光 |
| 江流进度线 | SVG / Canvas / Pixi Graphics | 水路节点，不做普通进度条 |
| 声音开关 | DOM 小题签 | `启声` / `静音`，不使用现代图标 |
| 重游入口 | DOM 小题签 | 水面光点式，不做普通按钮 |

## 七、开发质量红线

开发过程中出现以下情况，需要停下来回到本文调整，而不是继续堆效果：

- 首屏变成暗夜风，丢失 `design/第一轮/01.png` 的明亮雾青基调。
- 任何一幕出现普通网页导航栏、卡片套卡片、弹窗百科。
- 使用设计稿 PNG 作为最终整屏背景。
- 问诗接入在线 AI 或随机生成导致结果不可控。
- 尾声出现外链、二维码、社交分享入口。
- 素材加载依赖网络。
- 深色背景导致艾草、粽叶、编钟边缘出现明显白边或黑边。
- `bg-bamboo.webp` 被强行拉伸铺满横屏。

## 八、开发前检查清单

进入编码前，先确认：

- [ ] 每一幕参考的是对应 `design/第二轮` 静态稿。
- [ ] 每一幕三态参考的是对应 `design/第三轮` 分镜。
- [ ] 本文第四节的资产映射没有缺项。
- [ ] 所有文字最终用 DOM 或 Pixi Text 实现。
- [ ] `bg-bamboo.webp` 只作为局部层。
- [ ] `bg-river-dark.webp` 只作为后段低透明压暗层。
- [ ] 轻知识交互遵循随物解锁、触字显义、印记点亮。
- [ ] 所有资源路径为本地相对路径，不外链。

## 九、后续素材变更流程

如果某个素材需要替换：

1. 在 `assets/prompt-log.md` 追加生成记录。
2. 保持原文件名不变，避免开发代码大范围改路径。
3. 更新本文对应场景的“实现资产”和“已知限制”。
4. 在浅色背景和深色背景各检查一次透明边缘。
5. 重新跑对应页面的视觉验收。
