# 《楚江寻艾》提示词管理

## 一、管理目标

本项目计划使用 GPT 生成主要视觉素材，因此提示词需要像设计资产一样被管理。

目标不是“每次随手写 prompt”，而是建立一套稳定流程，保证生成素材具备：

1. **风格一致**：所有素材都属于同一套“雾青江光”的楚江端午视觉系统。
2. **主题准确**：每个素材都服务江、艾、粽、鼓、舟、诗、钟，不生成无关装饰。
3. **可复现**：每张入库素材都有可追溯的 prompt、版本和修改原因。
4. **易后处理**：精灵类适合抠图，背景类适合叠加，纹理类适合平铺。
5. **可落地**：生成结果满足 `素材管理.md` 中的尺寸、格式、体积和命名规范。

当前视觉锚点是 `design/01.png`：明亮雾青江面、远山云气、右侧暖金天光、左侧远舟悬念、前景清艾和中下部水纹入口。后续素材必须围绕这个基调延展，而不是回到全黑夜景。

## 二、生成平台与适配

本项目使用 **GPT 图像生成**（ChatGPT 图像生成、DALL-E 3 或 gpt-image-1）。

关键适配规则：

- **提示词统一使用中文**，生成画面中如需文字，也只允许出现指定中文。
- **色值只能作为方向**，生成结果需后处理校色。描述色彩时同时写“自然语言 + hex 参考”，例如“雾青江面，参考 #D8DED1 与 #9EAEA3”。
- **透明背景**：
  - 如果使用 gpt-image-1 API：设置 `background: "transparent"`，prompt 中仍写“透明背景”。
  - 如果在 ChatGPT 内直接生成：精灵类写“纯白背景，主体边缘干净，便于后续抠图”。
- **尺寸**：背景优先生成 16:9 横图；精灵根据主体生成方图、横图或竖图。
- **无缝纹理不保证**：纹理生成后必须用偏移检查接缝，必要时手动修补。
- **风格一致性不能只依赖种子**：必须依靠统一母版、参考图锚定、批量生成和后处理校色。

## 三、提示词总原则

所有提示词必须遵守以下规则：

- 先写用途，再写主体，再写风格，再写色彩，再写构图和背景要求。
- 每次只生成一种素材，不在同一张图里混合多个用途。
- 背景类强调留白、层次、雾青水面和可叠加空间，不抢交互主体。
- 精灵类强调孤立主体、边缘干净、背景易分离。
- 纹理类强调低对比、可平铺、无明显中心主体。
- 禁止生成水印、签名、边框、现代 UI、网页按钮、外链、二维码。
- 禁止恐怖、阴森、招魂、鬼怪、血迹等表达。
- 不追求“节庆热闹”，避免大红大金和商业国潮海报感。

统一风格描述：

```text
诗意东方，水墨长卷质感，雾青江面，远山云气，薄雾流动，暖金天光，克制电影感留白，安静沉浸，有一点实验交互感；参考 design/01.png 的明亮雾青江光，不要全黑夜景。
```

统一负面约束：

```text
不要生成多余文字、英文、水印、签名、logo、边框、二维码、现代建筑、现代 UI、普通网页按钮、卡片布局、霓虹赛博、卡通可爱风、恐怖元素、鬼怪意象、骷髅、血迹、暴力画面、全黑夜景。
```

## 五、提示词版本规则

每个素材使用独立编号，格式：

```text
[素材文件名] / [版本号] / [用途] / [日期]
```

示例：

```text
sprite-mugwort / v1 / 寻艾主精灵 / 2026-05-23
```

每次生成后必须记录：

- 使用的完整 prompt。
- 选中的图片编号或文件名。
- 保留原因。
- 淘汰原因。
- 后处理动作。
- 当前状态。

## 七、分类型 Prompt 模板

### 0. 标题艺术字

适用于首屏主标题、每幕大号章节字和极短装饰性动作短句。标题图只作为视觉层，不能替代 DOM 文本。

```text
请生成一张用于沉浸式交互网页《楚江寻艾》的透明背景中文标题艺术字。

文字内容：[替换为指定文案]。

视觉风格：诗意东方，水墨书写感，宋楷之间的笔意，雾青江光气质，克制、清润、安静，不要商业国潮海报感。笔触应自然、有手写温度，但字形必须清晰可读。

色彩：主笔触使用深墨绿 #16343A，可有少量雾白 #E8E1D2 或暖金 #C8A45D 的柔光边缘，但不要大面积金色、红色或高饱和绿色。

构图：只生成文字本身，透明背景，文字居中，四周留出 12% 安全边距，不能有边框、印章、图案、水纹、山水背景、英文、logo、水印或签名。

输出要求：透明 PNG 或 WebP，横向构图，适合叠加在雾青江面背景上。中文必须完全正确，不能多字、少字、错字、异体错用。
```

内置图像生成如不能直接输出透明背景，先生成纯色绿底，再用本地抠图工具转为透明资源。若生成结果出现繁简误用或错字，不能入库。

### 1. 精灵类

适用于艾草、鼓面、龙舟、编钟、粽叶、端午印记。

```text
请生成一张用于《楚江寻艾》的独立精灵素材。

主体：[一个独立主体]，位于纯白背景中央，主体边缘清楚，便于后续抠图。不要出现地面、桌面、投影、周围风景或其他物体。

视觉风格：诗意东方，水墨笔触，细节克制，材质柔和，和 design/01.png 的雾青江光风格一致。

色彩：[选择 2-3 个相关颜色，例如“艾草青 #8EA86A 为主，深墨绿 #1D332C 做叶脉和阴影，雾白 #E8E1D2 做轻微柔光”]。

构图：单个主体居中，轮廓清楚，缩小到 25% 后仍能识别，周围留白充足。

不要生成文字、水印、签名、logo、边框、场景背景、现代 UI、卡通图标、照片风、可爱风。
```

验收重点：

- 主体轮廓清楚，边缘适合抠图。
- 缩小到 25% 后仍能识别主体形态。
- 没有多余背景物、地面、桌面、投影。

### 2. 背景类

适用于入江雾青江面、江岸浅影、问诗深青水面、和鸣静水。

```text
请生成一张 16:9 横屏背景层，用于沉浸式交互网页《楚江寻艾》。

场景：[填写具体场景]。

视觉风格：诗意东方，水墨长卷质感，雾青江面，远山云气，薄雾流动，克制电影感留白，安静沉浸。整体参考 design/01.png 的明亮江光；如果是后段场景，可以让江面逐渐转深，但不能变成全黑夜景。

色彩：晨雾青 #D8DED1、浅水青 #9EAEA3、云水米白 #F1E8D2、雾白 #E8E1D2 为主要明亮层；楚江青黑 #101C1B、深墨绿 #1D332C 只用于水面深处和边缘阴影；必要时少量夕照暖金 #D9A65A。

构图：宽幅横向画面，中央或交互区域保留 60% 以上安静留白，不要强中心主体，适合叠加精灵、文字和水纹特效。

不要生成文字、人物、现代建筑、旅游照片感、节庆装饰堆叠、logo、水印、边框、全黑夜景。
```

验收重点：

- 不抢主体，中央或交互区域有足够留白。
- 适合在 1920×1080 下铺满。
- 明暗有层次，暗部不死黑，亮部不刺眼。

### 3. 雾气 / 遮罩类

适用于 `bg-fog-layer`、灰度遮罩、置换来源。

```text
请生成一张横向雾气遮罩素材，用于《楚江寻艾》的雾青江面。

画面只有稀疏、柔和、分层的横向薄雾，雾色为雾白 #E8E1D2 和云水米白 #F1E8D2。背景为纯黑或透明，便于后续作为 screen 混合或 alpha mask。

雾气要轻、薄、可流动，不要遮满画面，不要出现山、树、人、建筑、文字、水印、边框。
```

### 4. 可平铺纹理类

适用于水纹、端午暗纹。

```text
请生成一张可平铺低对比纹理，用于《楚江寻艾》的网页背景。

纹理主题：[水纹 / 云纹 / 艾叶暗纹 / 舟线节奏]。

视觉风格：水墨肌理，克制、安静、低对比，和雾青江面背景融合。没有明显中心焦点，没有单个大主体，重复铺满后应自然。

色彩：晨雾青 #D8DED1、浅水青 #9EAEA3、深墨绿 #1D332C、楚江青黑 #101C1B 的低对比组合；高光只用极少雾白 #E8E1D2。

不要生成文字、边框、水印、logo、强对比区域、亮色节庆符号、卡通龙、密集装饰。
```

### 5. 矢量参考类

适用于需要后续手工转 SVG 的轮廓参考。

```text
请生成一张用于手工描摹为 SVG 的简洁轮廓参考图。

主体：[叶片轮廓 / 舟身弧线 / 水纹圆环 / 印记外框]。
风格：极简、干净、带一点水墨笔触的自然弧线。
构图：主体居中，高对比纯色背景，轮廓简单，转 SVG 后控制在 20-40 个锚点。
不要生成复杂纹理、文字、阴影、水印、装饰边框。
```

### sprite-drumsticks 听鼓鼓槌

用途：第四幕“听鼓”第一视角鼓槌前景层，叠加在船头鼓面上，点击时做下压和回弹动画。

## 九、生成工作流

每个素材按以下流程执行：

1. **复制模板**：从本文选择对应类型和素材 prompt。
2. **生成 4 张变体**：同一 prompt 一次生成多版本，不立即改 prompt。
3. **初筛**：淘汰风格不符、构图不符、边缘复杂、颜色过亮或过暗的结果。
4. **定向迭代**：只改一个变量，例如“远舟更小”“江面更明亮”“轮廓更清楚”。
5. **入选记录**：记录最终 prompt、入选版本、淘汰原因。
6. **后处理**：按 `素材管理.md` 执行去背景、色彩校正、压缩。
7. **场景验证**：放入测试画布，在 `design/01.png` 对应的雾青江面和后段深青背景上分别确认效果。

### 分辨率策略

| 素材类型 | 建议生成尺寸 | 后处理目标尺寸 |
|----------|---------------|---------------|
| 精灵（方形） | 1024×1024 | 缩放到 512×512 |
| 精灵（长横形，如龙舟） | 1792×1024 | 裁切缩放到 1024×384 |
| 精灵（长竖形，如编钟） | 1024×1792 | 裁切缩放到 384×768 |
| 背景 | 1792×1024 或 16:9 横图 | 放大到 1920×1080 |
| 纹理 | 1024×1024 | 缩放到 512×512 |

### 风格一致性策略

1. **同一会话批量生成**：在同一个 ChatGPT 对话中连续生成同类型素材。
2. **参考图锚定**：首屏以 `design/01.png` 为锚点，后续 prompt 写明“延续这张图的雾青江光、远山云气和暖金天光”。
3. **统一后处理校色**：所有素材通过同一套曲线、色相饱和度和色彩平衡压入项目色板。
4. **先定锚再铺量**：先稳定入江背景和艾草精灵，再生成龙舟、粽叶、编钟等素材。

## 十、质量评审表

每张候选素材用 5 项打分，每项 1-5 分，低于 20 分不入库。

| 项目 | 评分标准 |
| --- | --- |
| 主题准确 | 是否一眼能看出对应端午/楚江元素 |
| 风格一致 | 是否符合雾青江光、水墨长卷、克制沉浸气质 |
| 构图可用 | 是否适合作为精灵、背景或纹理进入页面 |
| 后处理友好 | 是否容易抠图、压缩、裁切或平铺 |
| 技术适配 | 是否满足尺寸、透明度、体积、低对比等要求 |

淘汰常见原因：

- 太暗，背离首屏明亮雾青基调。
- 太亮，像节庆海报或旅游宣传图。
- 太写实，像照片素材。
- 太卡通，像小游戏图标。
- 元素太多，抢交互主体。
- 边缘太脏，抠图后有黑边或白边。
- 含有文字、水印、印章字或疑似 logo。
