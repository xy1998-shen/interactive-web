# 《楚江寻艾》提交说明

## 提交范围

最终提交包建议只包含运行作品必需文件：

- `index.html`
- `css/`
- `js/`
- `libs/`
- `assets/` 中运行清单实际引用的背景、精灵、纹理、字体、标题素材和 favicon
- `README.md`

不建议放入最终提交包：

- `.git/`
- `.omc/`
- `.claude/`
- `tmp/`
- `assets/source/`
- `design/`
- `doc/`
- `.DS_Store`

这些目录属于开发过程材料或源素材归档，会显著增加体积，不影响作品运行。

## 评审打开方式

推荐在桌面端 Chrome 或 Edge 中打开 `index.html`。作品以 PC 横屏为验收范围，最佳视口不低于 `960 x 540`，移动端暂不支持。

作品不访问远程接口、不加载外链资源、不依赖构建步骤。提交包解压后可直接打开入口文件。

## 评审路线

1. `入江`：轻触江面拨雾，阅读楚江端午引子。
2. `寻艾`：顺序浏览悬门、煮汤、入食、佩香，阅读自动展开的知识卡。
3. `裹青`：顺序浏览备料、裹粽、蒸煮、分食，阅读自动展开的知识卡。
4. `听鼓`：随鼓序列推进龙舟，轻点画面观察鼓波、水痕和船身惯性反馈。
5. `问诗`：触字成诗，进入楚辞意象。
6. `和鸣`：点亮编钟回声，连接诗声、风物和礼乐。
7. `端午印记`：点亮艾、粽、舟、钟四点，完成记忆收束。

## 验收状态

- PC 横屏主流程：已完成。
- 本地资源加载：已完成，入口和资源清单均使用相对路径。
- `file://` 兼容：按本地双击运行模型设计，使用普通脚本顺序和 UMD 全局对象。
- 移动端：暂不支持，不纳入当前提交验收范围。
